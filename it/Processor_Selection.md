## Selezione del processore

[MultiMarkdown](http://fletcherpenney.net/multimarkdown/ "MultiMarkdown") (di Fletcher Penney) è un'estensione di [Markdown](http://daringfireball.net/projects/markdown/ "Daring Fireball: Markdown") (di John Gruber). Viene rispettata tutta la formattazione Markdown standard e MultiMarkdown fornisce sintassi aggiuntive per elementi come tabelle e note a piè di pagina.

## Sintassi

C'è una guida alla sintassi Markdown nel menu Aiuto oppure puoi fare riferimento alla [documentazione originale di John Gruber](http://daringfireball.net/projects/markdown/syntax).

Le estensioni MultiMarkdown sono documentate da Fletcher Penney in [questo documento](https://github.com/fletcher/MultiMarkdown/blob/master/Documentation/MultiMarkdown%20User%27s%20Guide.md).

## Rendering alternativo

Puoi scegliere tra MultiMarkdown, Kramdown, CommonaMark e Sconto in {% prefspane Processor %}. Ciascuno offre funzionalità aggiuntive con una sintassi leggermente diversa (vedi {% appmenu Help, Markdown Reference %} per maggiori dettagli). Nella maggior parte dei casi, è preferibile MultiMarkdown, ma se disponi già di file che utilizzano la sintassi specifica del processore o riscontri problemi nel rendering di un file Markdown con MultiMarkdown, seleziona Commonmark come processore predefinito.
