# <%= @title %>

[Visual Studio Code] [vscode] non include Marked immediatamente, ma puoi utilizzare un'estensione della community per **anteprima Markdown live** in Marked --- anteprima, esportazione e correzione mentre continui a scrivere in VS Code.

## Avvio rapido

1. Installa un'estensione VS Code **Apri in contrassegnata** (vedi [Apri in estensione contrassegnata] [ext] di seguito).
2. Apri il file Markdown in VS Code.
3. Invia il file a Marked: l'anteprima si aggiorna quando salvi.

## Apri nell'estensione contrassegnata

L'[estensione Apri in Marked] [ext] (Visual Studio Marketplace) aggiunge un'azione **Apri in Marked**: pulsante del titolo dell'editor, **{% kbd shift cmd m %}**, menu contestuali nell'editor e in Esplora file, **apri cartella** opzionale per il browser dei file di Marked, un indicatore della barra di stato e salvataggio automatico opzionale prima dell'apertura. Le impostazioni ti consentono di impostare il percorso dell'app contrassegnata se non si trova nella posizione predefinita.

I> L'estensione è stata originariamente pubblicata per Marked 2. Marked 3 utilizza lo stesso stile di trasferimento di file e URL, quindi questa integrazione continua a funzionare; se qualcosa cambia, controlla la [pagina dell'estensione] [ext] o il repository dell'autore per gli aggiornamenti.

## Requisiti

Marked funziona solo su macOS. Installa [Marked 3] [marked] e l'estensione, quindi punta il **percorso dell'app** sull'app Marked, se necessario.
