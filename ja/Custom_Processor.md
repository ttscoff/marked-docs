### アクション [actions]

[*+ アクション*] ボタンを使用して、ルールにアクションを追加します。

![アクションを追加][追加アクション]

[addaction]: images/custom-rules-add-action-800.jpg @2x width=800

利用可能なアクションは次のとおりです。

スタイルの設定
: プレビューのスタイルを設定します。追加した組み込みスタイルまたはカスタム スタイルを使用できます。

コマンドの実行
: これは、引数を含むコマンド ライン コマンドを受け取り、ファイルの内容を STDIN で渡します。コマンドは結果の出力を STDOUT に返す必要があります。

> **Mac App Store サンドボックス**: Mac App Store (MAS) バージョンの Marked は、外部バイナリの実行を制限するサンドボックス環境で実行されます。 MAS バージョンで Pandoc などの外部プロセッサを使用する必要がある場合は、バイナリをアプリ バンドルにコピーする必要があります。これを行うには:
>
> 1. Marked.appを右クリック → パッケージの内容を表示
> 2. コンテンツ/リソース/に移動します。
> 3. `bin` フォルダーが存在しない場合は作成します
> 4. バイナリ (例: `pandoc`) をここにコピーします
> `bin` フォルダー
> 5. 実行可能にします: `chmod +x` バイナリ
> 6. 「コマンドの実行」アクションで、次のように参照します。
>
> `YOUR_BINARY_NAME [arguments]`
> またはフルパスを使用します:
> `/Applications/Marked.app/Contents/Resources/bin/YOUR_BINARY_NAME [arguments]`
>
> **注意**: 外部シバンを含むスクリプト (`/opt/homebrew/bin/python` を指す Python スクリプトなど) は、バンドルにコピーされるとシステム インタプリタを通じて自動的に実行されます。 MAS バージョンでは、バイナリ ファイルではなく、実際には Python または Ruby スクリプトであるバイナリの実行に依然として問題が発生する可能性があります。
> アップデートによりバンドル全体が置き換えられるため、アプリをアップデートするたびにバイナリを再コピーする必要があります。あるいは、**ヘルプ->クロスグレード**を使用して、そのような制限のない非サンドボックス バージョンを入手してください。

埋め込みスクリプトの実行
: 組み込みのコード エディターで完全なスクリプトを編集します。 Run Command と同様に、これは STDIN で入力を受け取り、STDOUT で出力を返す必要があります。

メタデータの設定
：メタデータを追加または設定します。キーと値を指定します。キーが存在する場合はその値が更新され、存在しない場合は追加されます。使用されるメタデータのタイプは、ファイルの内容 (またはメタデータ変換アクションの結果) によって自動的に決定されます。
: 既存のメタデータが見つからない場合、メタデータは HTML コメント内に MultiMarkdown 形式で追加されます。 Marked はこのメタデータを読み取ることができますが、どのプロセッサが使用されているかに関係なく、プレビューには表示されません。

メタデータの削除
: キーに基づいてメタデータを削除します。

メタデータの除去
: すべてのメタデータを削除します。 YAML、MultiMarkdown、Pandoc メタデータに影響します。

YAMLメタをMMDに変換
: ファイルの先頭にある YAML メタデータ ブロックを MultiMarkdown スタイルのメタデータに変換します。

MMDメタをYAMLに変換する
: MultiMarkdown メタデータ ブロックを YAML に変換します。

検索と置換
: ファイルの内容に対して検索と置換を実行します。
: 検索文字列がスラッシュで囲まれている場合 (例: `/Project \w+/`)、正規表現として扱われます。 `$1`、`$2` などを使用して、置換文字列に一致グループを含めることができます。
: 置換フィールドは、いくつかのエスケープ シーケンス (バックスラッシュの後に続く) をサポートしています。 `\n` は改行を挿入し、 `\t` はタブ文字を挿入し、 `\\` はリテラルのバックスラッシュを挿入し、 `\$` はリテラルのドル記号を挿入します (一致グループの代わりに)。
: その他の `\X` シーケンスは、単なる `X` (バックスラッシュが省略される) として扱われるため、`\e` は `e` になります。文字のない末尾の \ は、リテラルのバックスラッシュとして保持されます。
: 置換文字列で `[%key]` を使用して、ドキュメントのメタデータ、環境変数、またはコンテキストから値を挿入します (例: `[%title]`、`[%MARKED_PATH]`)。同じルール内の以前のアクションまたは前のルールによって設定されたキーが使用可能です。一致しないキーは空の文字列に置き換えられます。

タイトル H1 を挿入
: ドキュメントに H1 を挿入します。これは、メタデータまたはファイル名から取得できます。

シフトヘッダー
: ヘッダー レベルを -5 ～ +5 の範囲で調整します。たとえば、これを -1 に設定すると、すべての H1 が H2 になり、H2 が H3 になります。正の数に設定すると、ヘッダー レベルがその量だけ引き上げられます。

ヘッダーの正規化
: このアクションは、可能であれば、ドキュメント内に H1 が 1 つだけ存在することを保証し、すべてのヘッダー レベルを意味的な順序で調整し、レベルをスキップしないようにします。 H2からH4まで。元の文書がそもそも意味的に順序付けされている場合、これにより階層が適切に調整されます。

目次を挿入
: 目次を挿入します。 TOC は、最初の H1、最初の H2 の後、またはファイルの先頭に配置できます (メタデータの後に挿入されます)。

ファイルの挿入
: 選択したテキスト ファイルをドキュメント内の指定された位置に挿入します。これは、最初の H1、最初の H2、上部、下部、またはテキスト一致の後 (正規表現を検索するには `/pattern/` を使用します) にすることができます。

テキストの挿入
: アクションにテキストを直接埋め込むことができるポップアップ エディターを提供します。配置オプションは _ファイルの挿入_ と同じです。
: 挿入されたテキストの任意の場所で `[%key]` を使用して、ドキュメントのメタデータ、環境変数、またはマークされたコンテキストから値を取得します (例: `[%author]`、`[%MARKED_PATH]`)。これは、使用するプロセッサに関係なく機能するため、メタデータの置換に MultiMarkdown は必要ありません。同じルール内の以前のアクション (**メタデータの設定** など) または前のルールの値が含まれます。一致しないキーは空の文字列に置き換えられます。

CSSファイルを挿入
: 選択した CSS ファイルをドキュメントに挿入します。これはスタイルを選択した後にロードされ、既存のスタイルをオーバーライドしたり、新しいスタイルを追加したりするために使用できます。

CSSの挿入
: 独自の CSS をアクションに直接追加できるポップアップ CSS エディターを提供します。この CSS は、ドキュメントの上部、既存のスタイルの後に挿入されます。挿入されたスタイルの順序は、ルール内のアクションの順序と一致します。

JavaScriptファイルを挿入
: 選択した JavaScript ファイルをドキュメントの最後に挿入します。更新ごとにスクリプトを再読み込みする場合は、[更新フック](#updatehook) を指定して *Insert JavaScript* アクションを使用する必要があることに注意してください。

URLからJavaScriptを挿入
: これを使用して、CDN またはその他のリモート URL にリンクされた `<script>` タグをドキュメントの末尾に挿入します。更新ごとにスクリプトを再読み込みする場合は、[更新フック](#updatehook) を指定して *Insert JavaScript* アクションを使用する必要があることに注意してください。

JavaScript を挿入する
: カスタム JavaScript をアクションに直接埋め込むことができるポップアップ JavaScript エディターを提供します。これはドキュメントの最後に挿入され、ルールによって実行される JavaScript の順序はアクションのシーケンスによって決定され、最後のアクションが最後に追加されます。
注: 更新のたびにスクリプトを実行する場合は、[更新フック](#updatehook) を使用する必要があることに注意してください。

セルフリンク URL
: 裸の URL を `<url>` に変換すると、どのプロセッサでもハイパーリンクが生成されます。

ショートカットを実行
: Apple ショートカットを実行します。ショートカットを実行する場合は、STDIN から入力を受け取り、最後に出力を返す必要があります (停止して出力を返す)。テキストを変更しないアクションを実行する場合は、入力を変数に設定し、アクションを実行して、最後に元のテキスト変数を出力します。

## 環境変数 [environmentvariables]

「コマンドを実行」アクションには環境エディターがあり、
独自の環境変数を設定できます。
コマンドまたはスクリプトで使用できます。これらに加えて
カスタム変数、Marked には独自の変数がいくつか含まれています。

Marked は独自のシェルでカスタム プロセッサを実行します。つまり、
標準の環境変数は自動的には渡されません。
Marked の環境変数を使用して、
スクリプト内に独自のものを含めます。 Marked は次の変数を作成します
シェル スクリプトで使用できます。

**MARKED_ORIGIN**
: プライマリ作業ファイル (作業テキスト、Scrivener、またはインデックス ファイルのフォルダー) の場所 (ベース ディレクトリ)。

**パス**
: マークすると、デフォルトの実行可能フォルダーを含むパスが設定され、上記の `MARKED_ORIGIN` にディレクトリが追加されます。デフォルトは `/Developer/usr/bin:/usr/bin:/bin:/usr/sbin:/sbin:/usr/local/bin` です。必要に応じて PATH 変数を設定し、Marked のパス (例: `PATH=/usr/local/yourfolder:$PATH`) を追加または上書きすることで、独自のパスを追加できます。

**ホーム**
: ログインしているユーザーのホームディレクトリ。設定されている HOME 変数に依存する Python およびその他のスクリプトはこれを自動的に取得しますが、スクリプト内の他の用途にも使用できます。

**MARKED_EXT**
: 処理中のプライマリ ファイルの拡張子。この変数を使用すると、表示されているファイルの種類に基づいてさまざまなプロセスをスクリプト化できます。たとえば、`$MARKED_EXT == "md"` の場合は好みの Markdown プロセッサを実行しますが、`$MARKED_EXT == "textile"` の場合は Textile プロセッサを実行します。

**マークされたパス**
: これは、ロード時にマークで開かれているメイン ファイルへの完全な UNIX パスです。

**MARKED_INCLUDES**
: さまざまな [インクルード構文](Special_Syntax.html#pagebreaks) を使用して渡されるテキストにマークが含まれているファイルの、引用符で囲まれたカンマ区切りのリスト。

**MARKED_PHASE**
: これは「PROCESS」または「PREPROCESS」のいずれかに設定され、単一のスクリプトを使用してこの変数に基づいて両方のフェーズを処理できるようになります。

**MARKED_CSS_PATH**
: 現在のスタイルシートへのフルパス

### メタデータ環境変数 [metadata-environment-variables]

マークのコマンドで実行アクションが実行されるとき
Conductor システム、ドキュメントのメタデータは自動的に
抽出され、環境変数として利用可能になります。
コマンド。

#### 仕組み [how-it-works]

1. **メタデータ抽出**: システムは、以下をサポートする既存の `extractMetaDataFromString:` メソッドを使用してドキュメントからメタデータを抽出します。
   - YAML 前付 (`---` ブロック)
   - MultiMarkdown メタデータ (キー: 値行)
   - Pandoc メタデータ (`%` タイトル ブロック)
   - HTML コメントメタデータ (`<!-- key: value -->`)

2. **キーの正規化**: メタデータ キーは、環境変数として使用するために正規化されます。
   - 小文字に変換
   - 英数字以外の文字はすべて削除されます
   - スペースと特殊文字は削除されます

3. **環境変数の形式**: 各メタデータ キーは、接頭辞 `MD_` が付いた環境変数になります。
   - `Author` → `MD_author`
   - `XHTML header` → `MD_xhtmlheader`
   - `Custom Processor` → `MD_customprocessor`

#### 例 [example]

このメタデータを含むドキュメントがあるとします。

```markdown
title: My Document
author: John Doe
date: 2024-01-15
Custom Processor: true
Marked Style: amblin
XHTML header: <meta name="viewport" content="width=device-width">
Status: Draft
Priority: High
```

次の環境変数を使用できます。
コマンド:

```bash
MD_title="My Document"
MD_author="John Doe"
MD_date="2024-01-15"
MD_customprocessor="true"
MD_markedstyle="amblin"
MD_xhtmlheader="<meta name=\"viewport\" content=\"width=device-width\">"
MD_status="Draft"
MD_priority="High"
```

#### コマンドでの使用法 [usage-in-commands]

これらの環境変数を Run で使用できるようになりました。
コマンドアクション:

```bash
# Print the document title
echo "Processing: $MD_title"

# Use metadata in conditional logic
if [ "$MD_status" = "Draft" ]; then
    echo "Document is still in draft status"
fi

# Pass metadata to other tools
pandoc "$MARKED_PATH" \
  --metadata title="$MD_title" \
  --metadata author="$MD_author" \
  --metadata date="$MD_date" \
  -o output.pdf

# Use metadata for file naming
if [ -n "$MD_title" ]; then
    output_file="${MD_title// /_}.html"
else
    output_file="output.html"
fi
```
