# <%= @title %>

カスタム スタイル ジェネレーターは、CSS を手動で記述せずに Marked のカスタム スタイルを作成できる Web ベースのツールです。タイポグラフィ、色、間隔などのコントロールを備えたビジュアル インターフェイスと、変更を加えると更新されるライブ プレビューを提供します。

## ジェネレーターへのアクセス

スタイル ジェネレーターは [https://markedapp.com/style-generator/](https://markedapp.com/style-generator/)] で入手できます。 Web ブラウザで直接使用できます。インストールは必要ありません。

## はじめに

ジェネレーターを初めて開くと、次の内容が表示されます。

- **プレビュー ペイン** (左): サンプル マークダウン コンテンツに適用されたスタイルのライブ プレビュー
- **コントロール ペイン** (右): セクションに編成されたすべてのスタイル コントロール
- **ツールバー** (上部): スタイル タイトル、基本テーマ セレクター、および CSS インポート オプション

### 基本テーマの選択

まず、ドロップダウンから **ベース テーマ** を選択します。これによりスタイルの基盤が提供され、その後、スタイルのあらゆる面をカスタマイズできます。一般的なオプションには、空白 (最初から始める場合)、デフォルト、およびさまざまな組み込みテーマが含まれます。

### 既存の CSS のインポート

開始点として使用したい既存の CSS ファイルがある場合は、[**CSS のインポート**] をクリックしてファイルを選択します。ジェネレーターはこれらのスタイルをロードし、コントロールを使用してスタイルを変更できます。

## スタイルコントロール

ジェネレーターはコントロールを論理セクションに編成します。

### 基本タイポグラフィ

基本的なタイポグラフィ設定を制御します。

- **リズムを使用**: 有効にすると、一貫した見出しのサイズと間隔を確保するために数学的な活字スケールが使用されます。
- **ベース フォント サイズ**: ルート フォント サイズ (通常は 16px)
- **行の高さ**: テキストの行間の間隔
- **縮尺率**: 活版印刷の縮尺に使用される比率 (見出しのサイズに影響します)

### レイアウト

間隔とインデントを調整します。

- **ラッパー パディング**: コンテンツ領域の周囲のスペース
- **段落インデント**: 段落の最初の行のインデント
- **リストのインデント**: リストのインデント
- **ブロック引用インデント**: ブロック引用の左マージン

### フォント

フォント ファミリとウェイトを構成します。

- **ヘッダー フォント**: 見出し用のフォントのカンマ区切りリスト (例: "Georgia, serif")
- **本文フォント**: 本文テキストのフォントのカンマ区切りリスト
- **ヘッダーのウェイト**: 見出しのフォントのウェイト (100 ～ 900)
- **本文のウェイト**: 本文のフォントのウェイト
- **太字の太さ**: 太字テキストのフォントの太さ
- **文字間隔**: ヘッダーと本文の文字間隔

### Google フォント

Google フォントをスタイルに追加します。

1. 検索フィールドにフォント名を入力します (オートコンプリート候補が表示されます)。
2. オプションでスタイルを指定します (例: 通常、斜体、太字の場合は「400,400i,700」)
3. [**追加**] をクリックして追加します。
4. **Google フォントの参照** を使用して、プレビュー付きの完全なカタログを探索します

追加されたフォントはコントロールの下のリストに表示されます。フォントを削除するには、[×] をクリックします。

### 色

さまざまな要素の色を設定します。

- **背景**: ページの背景色
- **ベーステキスト**: デフォルトのテキストの色
- **ヘッダーの色**: すべての見出しの色 (見出しレベルごとに上書き可能)
- **本文の色**: 本文の文字の色
- **リンクの色**: リンクの色
- **強調色**: 強調されたテキストの色 (`<em>`)
- **強い色**: 強いテキストの色 (`<strong>`)
- **背景をマーク**: ハイライトされたテキストの背景色 (`<mark>`)

個々の見出しの色 (H1 ～ H6) は個別に設定できます。上書きをクリアしてヘッダーの色に戻すには、**リセット** を使用します。

### ダークモード

**ダーク モード**を切り替えて、ダーク モードのカラーをプレビューおよび構成します。有効にすると、ダーク モード バリアントの個別のカラー コントロールが表示されます。ダーク モード スタイルは、Marked の body 要素に `.inverted` が設定されている場合に適用されます。

**色の生成**を使用して、ライト モードのカラーに基づいてダーク モード パレットを自動的に作成します。

### 画像

画像の外観を制御します。

- **最大幅**: 画像の最大幅 (例: "100%"、"800px")
- **境界線の半径**: 丸い角 (例: "8px"、"0")
- **配置**: ドキュメントのデフォルト、左、中央、または右

### ブロック引用符

ブロック引用符のスタイルを設定します。

- **左境界線の幅**: 左境界線の太さ
- **左境界線の色**: 左境界線の色
- **背景色**: ブロック引用符の背景色
- **フォント スタイル**: 標準または斜体
- **左マージン**: 左端からの間隔
- **ネストされた左マージン**: ネストされたブロック引用符の間隔 (「継承」可能)

ダークモードのブロック引用符には個別のコントロールが使用できます。

### リスト

リストの外観を構成します。

- **リスト スタイルの位置**: 内側または外側 (箇条書き/数字が表示される場所)
- **左マージン**: 左端からの間隔
- **ネストされた左マージン**: ネストされたリストの間隔 (「継承」可能)

### 定義リスト

スタイル定義リスト (`<dl>`、`<dt>`、`<dd>`):

- **DL インデント**: 全体のインデント
- **DT** (用語) 設定: フォント サイズ、太さ、スタイル
- **DD** (定義) 設定: フォント サイズ、太さ、スタイル、インデント

### テーブル

包括的なテーブルスタイル:

- **背景色**: テーブルの背景
- **境界線の色**: セルの境界線の色
- **表のテキストの色**: 表のデフォルトのテキストの色
- **行/列を交互に**: カスタム カラーでゼブラ ストライプを有効にします
- **境界線**: テーブルのアウトラインの表示/非表示
- **グリッド**: 内部グリッド線の表示/非表示
- **配置**: 左または中央
- **境界線の半径**: 丸い角
- **最大幅**: テーブルの最大幅
- **セルのパディング**: 内部スペース
- **ヘッダー**設定: フォントの太さ、サイズ、スタイル
- **キャプション** 設定: フォントの太さ、サイズ、スタイル、テキストの色

ダーク モード テーブルには個別のコントロールが使用できます。

### コードブロック

コード ブロックとインライン コードのスタイルを設定します。

- **コード フォント ファミリー**: 等幅フォント スタック
- **背景**: 背景色
- **境界線の色**: 境界線の色
- **境界線の半径**: 丸い角
- **ラップ モード**: ラップなし (プリ)、保存 + ラップ (プリラップ)、または通常 (ラップ)
- **コード パディング**: 内部スペース

ダーク モード コード ブロックには個別のコントロールが使用できます。

### 脚注

脚注のスタイル:

- **マーカーの色**: 脚注マーカーの色
- **テキストの色**: 脚注のテキストの色
- **テキスト フォント スタイル**: 標準または斜体

ダークモードの脚注には個別のコントロールが使用できます。

### ドロップシャドウ

要素にドロップ シャドウを追加します。

1. シャドウ **強度**: ソフト、ミディアム、またはストロングを選択します。
2. シャドウを適用する要素を選択します。
   - 画像
   - コードブロック
   - ブロック引用符
   - テーブル

## カスタム CSS

使用可能なコントロールを超えた高度なカスタマイズを行うには、**カスタム CSS** ボタンを使用してコード エディターを開きます。ここで追加する CSS は、生成されたスタイルに追加され、Marked のドキュメント構造内で機能するように自動的にスコープ設定されます。

エディターには構文の強調表示と検証が含まれており、無効な CSS にはエラー メッセージが表示されます。

## ライブプレビュー

プレビュー ペインには、以下を含むサンプル マークダウン コンテンツに適用されたスタイルが表示されます。

- 見出し (H1 ～ H6)
- さまざまなインライン書式設定を備えた段落
- テーブル
- コードブロック
- 画像
- リスト (順序付きおよび順序なし)
- ブロック引用符
- 定義リスト
- 脚注
- タスクリスト

コントロールを調整すると、変更はリアルタイムで更新されます。

## 保存と共有

スタイルに満足したら、いくつかのオプションがあります。

### CSS を表示

[**CSS を表示**] をクリックすると、生成された完全な CSS がポップオーバーで表示されます。クリップボードにコピーすることも、保存する前に確認することもできます。

### CSS を保存

[**CSS を保存**] をクリックして、スタイルを CSS ファイルとしてダウンロードします。ダウンロードする前に、メタデータ (タイトル、作成者、説明) を入力するように求められます。

### マークに追加

[**マークに追加**] をクリックして、スタイルをマーク付きインストールに直接追加します。これには、Marked が実行されている必要があり、スタイル名と作成者情報を確認するダイアログが開きます。

### 共有スタイル

**スタイルを共有** をクリックして、他の人が使用できるようにスタイルを [マークされたスタイル ギャラリー](https://markedapp.com/styles) に公開します。以下を提供する必要があります:

- スタイルタイトル
- あなたの名前
- 著者のURL (オプション)
- 説明
- 注記 (オプション)

公開する前に、スタイルのプレビューが共有ダイアログに表示されます。

## メタデータ

メタデータ セクション (スタイル タイトルの横にある矢印ボタンで展開可能) を使用して、以下を設定します。

- **作成者**: あなたの名前 (セッション間で保持されます)
- **作成者 URL**: Web サイトまたはプロフィールの URL
- **説明**: スタイルの説明
- **注**: 追加の注記 (共有スタイルには含まれません)

このメタデータは CSS ファイルのヘッダーに含まれており、スタイルを共有するときに使用されます。

## ヒント

- 希望に近い基本テーマから始めて、カスタマイズします
- 最初から完全に制御したい場合は、**Blank** テーマを使用してください
- 数学的に一貫したタイポグラフィを実現するには **リズム** を有効にします
- サポートする予定がある場合は、**ダーク モード** トグルでスタイルをテストしてください
- **カスタム CSS** は控えめに使用してください。ほとんどのニーズは組み込みのコントロールで満たすことができます
- 共有する前に、さまざまなコンテンツタイプでスタイルをプレビューできます

## ブラウザの互換性

スタイル ジェネレーターは、最新のブラウザー (Chrome、Firefox、Safari、Edge) で最適に動作します。 JavaScript を有効にする必要があります。
