## 目次

![][8]

   [8]:images/tableofcontentsbutton.jpg @2x width=740px height=238px class=center

ドキュメントにヘッダーがある場合は、ツールバーにリスト ボタンが表示されます。クリックすると目次が展開されます。見出しをクリックすると、プレビューのそのセクションにジャンプします。 {% kbd j %}/{% kbd k %} (下/上) を使用してリスト内を移動し、{% kbd Enter %} または {% kbd o %} を使用して強調表示されたヘッダーにジャンプします。

**フィルター バー:** 目次を開いた状態で、{% kbd Space %} (スペース バー) を押して先行入力フィールドを開きます。見出し名の任意の部分を入力し (「マーク」では TextMate スタイルのマッチングが使用されます。たとえば、各単語の最初の文字を入力できます)、Tab ({% kbd ⇥ %}) または下矢印 ({% kbd ↓ %}) を押して、フィルターされたリスト内を移動します。

{% kbd cmd T %} を押すと、目次が開きます (または閉じます)。

{% prefspane General %} で ["見出しのセクションの折りたたみ"](Interface_Features.html#collapsibleheadlines) が有効になっている場合、Command キーを押しながら目次内の項目をクリックすると、そのセクションが折りたたまれたり展開され、必要に応じて親セクションが表示されます。

全画面モードの場合、目次はポップアップ メニューではなく固定サイド バーとして表示されます。通常のウィンドウ表示プレビューでそのレイアウトを使用するには、目次パネルの右下にある全画面切り替えを使用します。

   [12]: 画像/tableofcontentsfullscreentoggle.jpg @2x width=740px height=238px class=center


キーの要約リストについては、[キーボード ショートカット](Keyboard_Shortcuts.html#TableofContentsNavigation)を参照してください。

### 目次の全画面モード

マークされたプレビュー ウィンドウが全画面表示の場合、目次を左側に固定したままにして、常にナビゲーションを行うことができます。まだ {% kbd cmd T %} で切り替わります。このレイアウトでは、目次の外側をクリックしても目次が閉じられないことがよくあります。

通常のウィンドウで、目次パネルの下部にあるアイコンをクリックしてサイドバーとしてドッキングします。サイドバーの上部にあるアイコンをクリックしてポップアップ モードに戻します。

### 目次が表示される場所のカスタマイズ

[特別な構文](Special_Syntax.html#tocplacement) `<!--TOC-->` を使用して、エクスポートされたドキュメントに目次を挿入できます。

`max#` (例: `<!--TOC max2-->`) を追加して、表示される見出しレベルの数を制限します。

## 高速検索

**高速ナビゲーション** は、目次とフィルターに焦点を当てたものを組み合わせているため、最小限の入力でジャンプできます。

- プレビューで {% kbd f %} を押して、**フィルター フィールドがフォーカスされた**状態で目次を開きます (追加の手順を行わずに、目次を開いてから {% kbd Space %} を押すのと同じです)。
- 見出しタイトルの一部を入力します。リストは一致するものにフィルターされます。
- 見出しが 1 つだけ残っている場合は、Return ({% kbd ↩ %}) を押すと、その見出しに直接ジャンプします。
- 複数の見出しが残っている場合は、Tab ({% kbd ⇥ %}) を押してフィルターフィールドを離れ、{% kbd j %}/{% kbd k %} または矢印キーで移動し、{% kbd o %} または Return ({% kbd ↩ %}) を押して見出しに移動し、目次を閉じます。
- Tab キーを再度押すと、フォーカスが検索フィールドに戻ります。

> **ショートカット リマインダー:** 目次を開いて {% kbd Space %} を押すとフィルター バーが開きます。これは目次がすでに表示されている場合に便利です。

(以前のドキュメントではこれを「高速スイッチャー」と呼んでいました。これは同じ機能です。)

## ブックマークとミニマップ {#bookmarks-and-mini-map}

{% appmenu Gear %} プレビュー メニューを使用し、{% kbd Tab %} ({% kbd ⇥ %}) [検索](Interface_Features.html#search) の横にある文書にフォーカスを合わせて、流し読みしながらブックマークを配置して再アクセスします。

### ブックマークの設定

{% kbd shift 1 %}--{% kbd shift 9 %} を使用してスクロール位置にブックマークを設定し、{% kbd 1 %}--{% kbd 9 %} を単独で使用してジャンプして戻ります。 **ドキュメントの順序**で次/前に移動するには、{% kbd n %} と {% kbd p %} を使用します。 {% kbd shift n %} と {% kbd shift p %} は **数値** 順で次/前になります。

スタイルまたはページ サイズを変更すると、ブックマークが表示される場所が移動することがあります。ブックマークは一時的なレビュー補助として使用されます。ブックマークはドキュメント セッション間では保持されませんが、プレビューの更新や編集後は保持されます。

**見出しのブックマーク:** Option キーを押しながら {% kbd opt 1 %}--{% kbd opt 9 %} を押して、ビューポートの上部に最も近い見出し (または上部の前の最後の見出し) をブックマークします。

**次の空きスロット:** {% kbd cmd D %} (vim ユーザーの場合はバッククォート {% kbd \`\` %}) は、次に利用可能な番号付きスロットにブックマークを追加します。

{% kbd 0 %} を押して、ブックマーク ストリップ (見出しタイトルがある場合) を展開します。 [ミニマップ](#minimap)を有効にすると、{% kbd 0 %}で同時に表示されます。折りたたむには、Esc キーまたは {% kbd 0 %} をもう一度押します。

{% kbd x %} を 2 回 ({% kbd xx %}) 押して、すべてのブックマークをクリアします。

[その他のプレビュー ショートカット](Keyboard_Shortcuts.html) があります。プレビューで {% kbd h %} を押してヘッドアップ リストを表示するか、{% kbd opt cmd K %} を押して完全なリファレンスを表示します。

### ミニマップ {#minimap}

{% prefspane Preview %} 設定でミニマップが有効になっている場合、{% kbd 0 %} はブックマーク ストリップに沿ってドキュメント全体の拡大縮小されたサムネイルを開きます。マップ上の任意の場所をクリックすると、そこで完全なプレビューがスクロールされます。保存されたブックマークは、番号 (関連する場合は見出し) の付いた水平線として表示されます。

**コマンド**を押したままミニマップ上を移動すると、拡大されたルーペが表示されます。 **オプション**を押したままドラッグして、スクロールバーをドラッグするようにスクロールします。

ウィンドウのサイズやレイアウトが変更されると、マップが再生成されます。非常に長い文書の場合、{% kbd 0 %} を 1 回押すと少し時間がかかる場合があります。 Marked は、要求するまで、初期ロード時にミニマップが自動的に構築されることを回避します。

{% kbd 0 %} または Esc キーを押してミニマップを閉じます。

**パフォーマンスに関する注意:** マップを生成すると、巨大なドキュメントのプレビューが一時停止することがあります。これは、マップが表示されているとき、またはサイズ変更後にのみ実行されます。
