# <%= @title %>

Mac の [下書き][下書き] は、**マークされたストリーミング プレビュー** ([ストリーミング プレビュー](Streaming_Preview.html) で説明したのと同じクリップボード ベースのチャネル) を使用して、アクティブな下書きをマークにミラーリングできます。コミュニティ **アクション**を使用して現在のドラフトを 1 回送信することもできます (再度実行するまでライブ更新は行われません)。

## ストリーミング プレビュー (Mac 用の下書き)

1. [マーク] で {% appmenu File, New, Streaming Preview %} を選択すると、ストリーミング プレビュー ウィンドウの準備が整います。
2. **Drafts for Mac** で、**設定**を開き、**マークされたアプリのストリーミング プレビュー サポートを有効にする**を有効にします。
3. 下書きでマークを前に出すことができる場合は、**マークを開く** を使用し、下書きで編集します。下書きが変更されるとプレビューも更新されます。

![][draftsprefs]

このオプションがオンの場合、Drafts は、Marked が (ディスク上のファイルの監視に依存するのではなく) ストリーミング プレビュー用に公開する統合を介して、Marked に Markdown をプッシュします。

### マークされる

Drafts の設定シートに **Get Marked App** が表示されている場合は、Marked がまだインストールされていない場合に使用してください。

## マークされたアクションでのプレビュー (手動更新)

下書きディレクトリからコミュニティ アクション [**マーク付きプレビュー**](https://actions.getdrafts.com/a/11f) をインストールします。 **現在の下書きテキスト**を、`x-marked://preview?text=…` URL を使用して Marked に送信します (下書きの内容は下書きに置き換えられます)。 **コンテンツはライブ更新されません**: マークを下書きと一致させたいときはいつでも、アクションを再度実行します。

このアプローチは、**ストリーミング プレビュー** が継続的な書き込みセッションに適している一方で、時折チェックする場合に便利です。

Draft は iPhone や iPad でも実行できます。ここに記載されているストリーミング プレビューの統合は、同じ Mac 上のマーク付き **Drafts for Mac** に適用されます。
