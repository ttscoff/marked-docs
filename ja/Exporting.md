## プレビュー後にエクスポート

Marked のプレビューはエクスポートの基礎です --- プレビュー ウィンドウに表示される内容は、PDF、DOCX、EPUB、およびその他の形式 (マージン、ヘッダー、ページネーションなどのモジュロ エクスポート固有の設定) で得られるものです。まずスタイルを設定してプレビューで校正し、ドキュメントの準備ができたらエクスポートします。完全なプレビュー ワークフローについては、[Mac でのライブ Markdown プレビュー](Live_Markdown_Preview_on_Mac.html) を参照してください。

## エクスポート パネル [ドロワー]

![エクスポートパネル][エクスポートパネル]

**エクスポート パネル** は、キーボード駆動のスポットライト スタイルのパネルで、すべてのエクスポート オプションに素早くアクセスできます。ステータス バーのエクスポート アイコンをクリックするか、{% kbd shift cmd e %} を押して開きます。

![エクスポートボタン][expbut]

エクスポート パネルを使用すると、ドキュメントを HTML、単一ページ PDF、ページ分割された PDF、RTF パッケージ、または Microsoft Word DOC または DOCX ファイルとして保存できます。新しい Markdown ファイル (マークされた固有の機能がレンダリングされ、その結果が含まれます)、Open Document (ODT)、または他のアプリケーションで使用する OPML として保存することもできます。

## HTML をコピー

HTML のコピー機能を使用すると、手間をかけずにプレビューの HTML ソース コードをクリップボードに保存できます。歯車メニューから選択するか、{% kbd shift cmd C %} を押すだけです。コピーされた HTML は、ブログ、フォーラム、または HTML ドキュメントにすぐに挿入できるスニペットになります。

コピーするためにソース ビューにいる必要はありません。プレビューにフォーカスを当てて (プレビューをクリックして)、「{% kbd shift cmd C %}」と入力すると、ソースがクリップボードにあることを知らせるポップアップ メッセージが表示されます。

## HTML を保存

![][exporthtmlaccessory]



[HTML を保存] コマンド (歯車メニューからアクセスするか、**{% kbd cmd S %}** と入力するだけでアクセスできます) を使用すると、共有または公開できる完全なドキュメントを保存できます。

オプションで、Marked のスタイル (または [カスタム スタイル][カスタム] の 1 つ) をエクスポートに含めることができ、必要な書式設定がすでに埋め込まれたすぐに使えるドキュメントを作成できます。

さらに、ドキュメントに含まれるローカル画像をエクスポートされた HTML 内に埋め込むことを選択できるため、画像を一緒に移動することなく、どこにでもホストできるスタンドアロンのドキュメントを作成できます。これは、ローカル ドライブ上で相対パスまたは絶対パスを使用して参照されるイメージでのみ機能します。この機能を利用したい場合は、`file:///` パスの使用を避けてください。

## Mac で Markdown を PDF にエクスポート

印刷/PDF プレビュー ({% kbd cmd P %}) は、標準の印刷ダイアログを表示します。 Marked の各プレビュー スタイルには、背景を削除し、文字サイズを変更し、境界線を提供する独自の付随印刷スタイルがあります。プレビューは、現在選択されているスタイルに基づいて印刷されます。

印刷ダイアログ ボックスで目立つのは、[PDF] ボタンと [プレビュー] ボタンです。 PDF では (使用可能なアプリケーションに基づいて) PDF にエクスポートするためのさまざまなオプションが提供され、プレビューでは PDF バージョンを Preview.app に直接エクスポートし、保存したり電子メールで送信したりできます。

この方法を使用して PDF に印刷すると、ページネーションが含まれます。紙または PDF に印刷する場合、[`<!--BREAK-->` 構文][break] を使用するか、{% prefspane Export %} でレベル 1 および/またはレベル 2 のヘッダーをセクション区切りとして使用するオプションを設定することにより、改ページを手動で挿入できます。

印刷時に水平罫線 (`<hr>`) を改ページに変換する設定もあります。これを行うと、タグによって作成された行が改ページに置き換えられ、最終出力から削除されます。プレビューはこの設定の影響を受けません。

![印刷マージン][印刷マージン]

ページ余白は{% prefspane Export %} で設定でき、印刷およびページ分割された PDF 出力に影響します。

`Margins:` メタデータ キーを使用して、ドキュメントごとの余白設定を上書きできます。値はポイントとして解釈されます。 `px`、`pt`、`em` などの単位接尾辞は無視されます。垂直方向と水平方向の余白には 2 つの数字 (`10 20`) を使用するか、上、右、下、左に 4 つの数字 (`10, 20, 10, 20` または `10 20 10 20`) を使用します。メタデータのマージンは {% prefspane Export %} 設定をオーバーライドします。

### ヘッダーとフッター

{% prefspane Export %} で定義されたヘッダーとフッターは、印刷またはページ分割された PDF に保存されたページ、および DOCX エクスポートのページの上部と下部に表示されます。左上、中央上、右上、左下、中央下、右下に任意のテキストを配置できます。中央のテキストはページの中央に配置されます。次の変数を使用すると、文字列内で置き換えられます。

    %title = ドキュメントのタイトル
    %date = 現在の日付
    %time = 現在の時刻
    %page = 現在のページ番号
    %total = 合計ページ数
    %path = ドキュメントへのファイルシステムのパス
    %filename = ドキュメントのファイル名のみ
    %basename = 拡張子なしのファイル名
    %logo/%image = ヘッダー/フッター設定で画像に設定されたロゴ
    %%(text) = 最初のページにのみ印刷するテキスト
    %h1/h2/h3/h4/h5/h6 = Markdown ヘッダーに基づくセクション タイトル
    %sep(X) = サブセクションが存在する場合にセクション タイトルの間に配置するテキスト

**印刷およびページ分割された PDF** は、Marked のページ分割を使用して `%h1`--`%h6` を解決するため、各ページにはそのページに表示される見出し階層が表示されます。 `%sep(X)` および `%md_*` メタデータ変数も印刷/PDF 出力でサポートされています。

**DOCX エクスポート** は、Word ヘッダーまたはフッターに `%logo`/`%image` を埋め込みます (高さ約 50 ピクセルに拡大縮小され、印刷プレビューに一致します)。見出しプレースホルダーは、エクスポートされた `Heading 1`--`Heading 6` スタイルを参照する Word **STYLEREF** フィールドになります。 Word は、Marked のプレビュー ページネーションではなく、Word 独自のレイアウトと改ページに基づいて、文書を開いたときにこれらのフィールドを更新します。 `%md_*` メタデータ変数は、印刷/PDF の場合と同様に、エクスポート時に 1 回置換されます。 `%sep(X)` は DOCX ヘッダー/フッターでは変換されません。

`%title` は、MultiMarkdown メタデータ ヘッダーで定義されている「Title:」情報を使用します。それ以外の場合は、ファイル拡張子なしのファイル名に戻ります。 MMD メタデータを使用してタイトルを定義するには、ドキュメントの最初の行に次の行を配置します。

    タイトル: ドキュメントのタイトル

この行の後に空行を続けます (または、定義したい他のメタデータの後に空行を続けます)。

また、MMD メタデータ キーに `%md_` をプレフィックスとして付け、キーの単語を小文字の文字列として組み合わせることで、変数として使用することもできます。たとえば、ドキュメントの先頭に次のようなメタデータ キーがあるとします。

    ファンキー モンキー: 最もファンキーな猿

次に、ヘッダー フィールドで `%md_funkymonkey` を使用すると、エクスポートされたドキュメントのヘッダーの場所に「The funkiest Monkey」が挿入されます。特定のキーが含まれていないドキュメントではフィールドが空白のままになり、メタデータに基づいてヘッダーを選択的に追加できます。

`%date` および `%time` 形式コードについては、[日付と時刻の形式](Exporting.html#dateandtimeformats) を参照してください。

「ページ番号のオフセット」設定を使用して、ページ番号が何番目から始まるかを調整できます。たとえば、最初のページに目次があり、2 ページ目から番号を 1 ページとして開始する場合は、オフセットを -1 に設定します。ヘッダー/フッター (`%page`) ではページ 2 がページ 1 になり、それに応じて合計ページ (`%total`) が調整されます。

ファイルの先頭にある MMD メタデータを使用して、特定のドキュメントのヘッダー/フッター フォントを指定することもできます。

    ヘッダーのフォント: Mensch

フォント ファミリ名を使用すると、デフォルトのフォントフェイスが選択されることに注意してください。フォントのシステム名を使用してバリエーションを指定できます。たとえば、Helvetica Neue Ultralight の場合は、「ヘッダー フォント: HelveticaNeueUltralight」を使用します。

さらに、メタデータを使用してドキュメントごとのヘッダー/フッターのフォント サイズを指定できます。

    ヘッダーのフォント サイズ: 10

### 日付と時刻の形式

{% prefspane Export %} の **日付形式** フィールドと **時刻形式** フィールドは、ヘッダーとフッターで `%date` と `%time` がどのように表示されるかを制御します。どちらのフィールドも strftime 形式のコード (`%` の後に文字が続く) を使用します。リテラル テキスト (`-`、`:`、スペースなど) はそのままコピーされます。

**例**

    %m-%d-%Y → 06-20-2026 (マークのデフォルトの日付形式)
    %I:%M %p → 3:45 PM (Marked のデフォルトの時刻形式)
    %Y-%m-%d → 2026-06-20
    %B %d, %Y → 2026 年 6 月 20 日
    %a %H:%M → 金 15:45

**共通フォーマットコード**

|コード |例 |説明 |
| ---- | ------- | ----------- |
| `%Y` | 2026年 | 4 桁の年 |
| `%y` | 26 | 2 桁の年 |
| `%m` | 06 |月 (01--12) |
| `%B` | 6月 |完全な月名 |
| `%b` |ジュン |月の略称 |
| `%d` | 20 |月の日 (01--31) |
| `%e` | 20 |日付 (スペース埋め込み) |
| `%A` |金曜日 |完全な曜日名 |
| `%a` |金 |曜日の略称 |
| `%H` | 15 |時、24 時間制 (00--23) |
| `%I` | 03 |時、12 時間制 (01--12) |
| `%M` | 45 |分 (00--59) |
| `%S` | 07 | 2 番目 (00--59) |
| `%p` |午後 |午前または午後 |
| `%x` | (ロケール) |ロケールの希望日 |
| `%X` | (ロケール) |ロケールの希望時間 |
| `%c` | (ロケール) |ロケールの希望日時 |

**印刷およびページ分割された PDF** は、上記の完全な strftime スタイル セットに加えて、`%j` (日付)、`%U`/`%W` (週番号)、`%z` (タイムゾーン オフセット)、および `%Z` (タイムゾーン名) などの追加コードをサポートします。完全なリファレンスについては、[Open Group strftime 仕様](https://pubs.opengroup.org/onlinepubs/9699919799/functions/strftime.html) を参照してください。

**DOCX エクスポート** は、上の表にリストされているコードをサポートしています。あまり一般的ではないコードは無視されるか、変更されずに渡される場合があります。

{% prefspane Export %} の **デフォルト形式の復元** を使用して、`%m-%d-%Y` と `%I:%M %p` にリセットします。

### ドキュメントごとのヘッダーとフッター

ドキュメントの最上部にある MultiMarkdown メタデータを使用して、ドキュメントごとにヘッダーとフッターを指定できます。

    ヘッダーを左に印刷: %title
    印刷ヘッダー センター: %basename
    ヘッダーを右に印刷: %date
    フッターを右に印刷: %page/%total

これらは環境設定の設定をオーバーライドします。 MultiMarkdown 以外のプロセッサを使用していて、ヘッダーをドキュメント自体に表示したくない場合は、HTML コメントを使用して、コメントを閉じる前に必ず空白行を残すことができることに注意してください。

    <!--
    ヘッダーを左に印刷: %title
    印刷ヘッダー センター: %basename
    ヘッダー右を印刷: %date

    -->

## PDF を保存

このオプションでは、プレビューをドライブ上の PDF ファイルに直接保存します。ドキュメントは改ページせずに全体がレンダリングされます。出力にページネーションを含めるには、[エクスポート パネル](#drawer) の印刷/PDF オプションを使用します。

## RTF エクスポート オプション

Marked では、RTF (リッチ テキスト フォーマット) データをクリップボードに直接エクスポートできます。歯車メニューから [リッチ テキストのコピー] コマンドを選択するだけです。

Marked では、ファイルを **RTFD** (リッチ テキスト形式) ファイルとして保存することもできます。 RTFD 形式は、RTF ファイルとドキュメントに埋め込まれた画像を含むバンドル形式です。

## DOCX エクスポート

DOCX としてエクスポートすると、見出し、ヘッダー/フッター、強調、リストなどの要素がすべて有効な Word スタイルにマップされた、有効な Microsoft Word ドキュメントが作成されます。つまり、Word で独自のテーマを簡単に適用できます。

Word のインポートとエクスポートの詳細については、[DOCX の操作][DOCX] を参照してください。

## マークダウンを EPUB にエクスポート

Marked は、100% 有効で 100% アクセス可能な EPUB ドキュメントをエクスポートできます。 EPUB エクスポート タイプを選択し、タイトル、作成者、日付などのメタデータを指定し、オプションでカバー写真を追加します。保存されたファイルは、EPUB ビューアで読み取ることができます。

EPUB エクスポートに必要なメタデータは、Markdown ドキュメント、Scrivener ファイル (`metadata` ページを含む)、またはその他の書籍形式であっても、ファイル自体に含めることができます。使用するキーは次のとおりです。

```yaml
title: Your Document Title
author: Your Name
cover image: path/to/image.jpg
```

Cover Image キーには、ベース ドキュメントに対する相対パスまたは絶対パスを含めることができます。 PNG または JPG ファイルを使用できます。

タイトルが設定されていない場合は、デフォルトで元のドキュメントのファイル名が使用され、作成者が設定されていない場合は、デフォルトで macOS ユーザーのフルネームが使用されます。

日付は常に現在の日付に設定され、現在メタデータを使用して変更することはできません。ただし、フォーマット (ISO) が変更されていない限り、保存時に変更できます。

### Amazon Kindle (KDP) への出版

EPUB は、Kindle だけでなく、多くの読書アプリやストア (Apple Books、Kobo など) で使用されているオープン形式です。 [Kindle ダイレクト パブリッシング (KDP)](https://kdp.amazon.com/) を通じて出版する場合は、Marked から EPUB をエクスポートし、そのファイルを KDP にアップロードします。 Amazon は、読者向けにそれを独自の Kindle 配信形式 (KFX) に変換します。

KDP は、EPUB や DOCX (Marked はエクスポートも可能) など、いくつかの原稿形式を受け入れます。要件については、Amazon の [サポートされている電子書籍形式](https://kdp.amazon.com/en_US/help/topic/G200634390) および [電子書籍原稿フォーマット ガイド](https://kdp.amazon.com/en_US/help/topic/G200645680) を参照してください。

**MOBI は必要ありません。** KDP は、新しいタイトルの MOBI アップロードを受け付けなくなりました (2025 年 3 月の時点で、固定レイアウトの書籍を含む)。 Marked は MOBI をエクスポートしないため、KDP 用の別個の「Kindle」または Mobipocket ファイルは必要ありません。 Amazon のレイアウト ツールを使用したい場合は、KPF ファイルを生成する [Kindle Create](https://kdp.amazon.com/help/topic/G8UZKZD45RTQNNVT) を使用して本を準備することもできます。

アップロードする前に、Amazon の無料の [Kindle Previewer](https://kdp.amazon.com/help/topic/G202131170) を使用して、EPUB が Kindle 端末でどのように表示されるかを確認するとよいでしょう。これは Amazon のオプションのサードパーティ ソフトウェアであり、Marked の一部ではありません。

## プロファイルのエクスポート

エクスポート プロファイルを使用すると、エクスポート設定のさまざまなセットを保存し、すばやく切り替えることができます。これは、さまざまな目的でドキュメントを定期的にエクスポートする場合に特に便利です。たとえば、特定のマージンとヘッダーを備えた印刷用 PDF 用のプロファイルと、スタイルが埋め込まれた Web 用 HTML 用のプロファイルなどです。

### エクスポート プロファイルの使用

初めて Marked を起動すると、現在のエクスポート設定を使用して「デフォルト」プロファイルが自動的に作成されます。ダイアログの上部にあるプロファイル ポップアップ メニューを使用して、エクスポート ダイアログ (PDF エクスポート、HTML の保存など) でプロファイルを表示および選択できます。

**新しいプロファイルの作成:**

1. {% prefspane Export %} または任意のエクスポート ダイアログでエクスポート設定を調整します。
2. エクスポートダイアログで、プロファイルポップアップメニューをクリックし、「プロファイルの追加…」を選択します。
3. プロファイルの名前を入力します (例: 「印刷品質」または「Web エクスポート」)
4. 現在の設定がそのプロファイルとして保存されます

**プロファイルのロード:**

- エクスポート ダイアログのポップアップ メニューからプロファイルを選択します
- すべてのエクスポート設定は、そのプロファイルの保存された値と一致するようにすぐに更新されます。

**プロファイルへの変更の保存:**

- エクスポート設定を変更すると、プロファイルポップアップの横に「保存」ボタンが表示されます。
- [保存] をクリックして、変更内容を現在のプロファイルに更新します。
- ボタンは未保存の変更がある場合にのみ有効になります。

**プロファイルの管理:**

- プロファイルポップアップメニューから「プロファイルの管理…」を選択して、プロファイル管理ウィンドウを開きます。
- そこから次のことができます。
  - **プロファイルの名前を変更**するには、名前をダブルクリックします
  - **プロファイルを複製**して、それに基づいて新しいプロファイルを作成します
  - **プロファイルを削除** (「デフォルト」プロファイルは削除できません)
  - 利用可能なすべてのプロファイルをリストで表示

### エクスポート プロファイルがキャプチャするもの

エクスポート プロファイルには、次のようなエクスポート関連の設定がすべて保存されます。

- **PDF 設定**: ページ サイズ、余白、ヘッダー/フッター、フォント、改ページ、目次
- **HTML エクスポート**: スタイルの埋め込み、画像の埋め込み、構文の強調表示、数式レンダリング
- **マークダウン処理**: テキストの折り返し、リンクの書式設定、プロセッサ ルール
- **CriticMarkup**: エクスポートの種類と処理オプション
- **構文の強調表示**: 言語の検出と強調表示の設定
- **数学レンダリング**: MathJax/KaTeX の統合と方程式の番号付け
- **画像処理**: 埋め込みオプション、コピー動作、パス設定
- **タイポグラフィ**: ハイフネーション、未亡人/孤児、フォント サイズ
- **エクスポート動作**: ファイルの命名設定、競合解決

プロファイルは、マークダウン、HTML、連続 PDF、ページ分割された PDF、EPUB、DOCX、ODT、TextBundle、RTF、OPML など、すべてのエクスポート タイプで機能します。

### プロファイルストレージ

プロファイルは、次の場所にある Application Support フォルダーに保存されます。

    ~/ライブラリ/アプリケーションサポート/マーク/ExportProfiles.plist

つまり、アプリの設定をリセットした場合でもプロファイルは保持され、アプリの更新後もプロファイルは存続します。このファイルをバックアップして、インストール間でプロファイルを保存できます。

### エクスポート プロファイルの使用に関するヒント

- **一般的なワークフローのプロファイルを作成**: 印刷用と Web 用に定期的にエクスポートする場合は、それぞれに個別のプロファイルを作成します。
- **わかりやすい名前を使用します**: 「印刷 - レター」や「Web - 埋め込みスタイル」などのプロファイル名を使用すると、各プロファイルの用途が明確になります。
- **頻繁に保存**: 変更を加えるたびに [保存] ボタンが表示されます。調整を保存するにはクリックしてください。
- **既存のプロファイルから開始**: 管理ウィンドウの「複製」を使用して、最初から開始するのではなく、既存のプロファイルのバリエーションを作成します。
