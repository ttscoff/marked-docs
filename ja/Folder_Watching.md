# <%= @title %>

プレーンテキストのメモの **フォルダー** を Marked にドロップします。 Marked は、そのフォルダー内の **最近編集された**対象ファイル (Marked のファイル タイプ フィルターに一致する .md、.txt、.markdown など) を常に追跡するプレビュー ウィンドウを開きます。

監視されているファイルが保存されるたびに更新が実行されます。最新のファイルが前のファイルと一致する場合、マークは検出された編集領域に向かってスクロールします。ドキュメントを切り替えると、プレビュー全体がアクティブなファイルに切り替わります。

## nvUltra、nvALT、および同様のツールとうまく連携します

個々のファイルをディスク上に残すノートブック アプリ (従来の **nvALT**、**nvUltra**、**Notational Velocity** スタイルのライブラリ、同期された Git フォルダー、Dropbox スクラッチ フォルダーなど) は、フォルダー監視と自然に組み合わされます。手動でプレビューを再度開くことなく、1 つのウィンドウで書き込み、その横にマークを固定したままにすることができます。

**nvUltra** では、上記で説明したフォルダー全体のウォッチャー フローにマークをアタッチする代わりに、マーク内の特定のノートを直接開く場合、コンテキスト メニューに **[マーク内のファイルをプレビュー](nvUltra.html)** も提供されます。

また、Marked は、多くの小さな章を 1 つの読書体験のように感じるものに統合するときに、他のメニュー名でも同じウォッチャーの動作を公開します。複数の Markdown ソースを意図的に結合した原稿については、[マルチファイル ドキュメント](Multi-File_Documents.html) も参照してください。
