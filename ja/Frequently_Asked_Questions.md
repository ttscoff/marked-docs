## ライセンス コードはすでに使用されています [license_code_has_already_been_utilized]

Marked 2 を新規インストールし、ライセンスの入力時に「ライセンス コードはすでに使用されています」というエラーが表示された場合は、<a href="mailto:me@brettterpstra.com">私に連絡</a>して新しいライセンスをリクエストしてください。 **登録した電子メール アドレスおよび/または現在のライセンス コードを入力してください。**

Marked 用に生成された初期のライセンスには、マシン制限ではなく使用制限があったため、同じマシン上であっても 3 回インストールするとアクティベーションが消費されてしまいます。これらの制限は、最近生成されたライセンスで修正されています。 Marked 2 ライセンスを購入すると、所有するどのマシンにも Marked 2 をインストールできるため、問題が発生した場合は、遠慮なく交換を依頼してください。

## URL が検証されないか、「不明」が返される [a_url_wont_validate_or_returns_unknown]

Marked の [リンク検証](http://marked2app.com/help/Link_Validation.html) は、基本的な HEAD リクエストを使用して、リンクが有効かどうかを判断します。 200 (成功) 以外の結果は、不明、または 404 (見つからない) や 500 (サーバー エラー) などの一般的なエラー コードの場合はエラーになります。認証の背後にある URL (Apple Developer URL や、アクセスにログインが必要なものなど) は「不明」を返します。また、サーバーが奇妙な応答コードを返す Amazon.com などの特定のサイトも同様です。これに関してマークができることはあまりない。

## MAS バージョンのカスタム プロセッサー権限 [custom_processor_permissions_in_mas_version]

サンドボックスの制限により、Mac App Store バージョンの Marked 2 は、特定の種類のバイナリ ツールをカスタム プロセッサとして実行できません。この制限に遭遇した場合は、いくつかの手順を試すことができます。

1. **詳細** ペインの **マーク 2** 設定 (⌘,) に移動し、[権限の更新] をクリックしたことを確認します。これにより、マークされたユーザーにデフォルトのドライブ全体へのアクセスが許可され、一時フォルダーやデフォルト以外の場所にアクセスする必要があるスクリプトやユーティリティの問題が解決されます。
2. スクリプトを使用してみます。実行するユーティリティ (multimarkdown、kramdown など) をシェル スクリプトで参照します。 bash、Ruby、Perl、または Python スクリプトを使用できます。次に、詳細設定でプロセッサを関連するシェルまたは実行可能ファイルに設定し、パラメータをスクリプトの場所に設定します。たとえば、~/scripts/mmd_wrapper.sh に保存される bash スクリプトを作成できます。
    
        #!/bin/bash
        猫 | 猫/usr/local/bin/multimarkdown
    
    次に、それを実行可能にし (`chmod a+x ~/scripts/mmd_wrapper.sh`)、カスタム プロセッサーの設定を行います。

        プロセッサ: /bin/bash
        引数: /Users/me/scripts/mmd_wrapper.sh
3. 一部の実行可能ファイル (特に Pandoc) はサンドボックス内では動作しません。この場合、実行時にポップアップ表示されるエラー ウィンドウから連絡して、ダイレクト バージョンへのクロスグレード ライセンスを取得してください。

## エクスポートされた PDF 内のドキュメント内リンク [intra-document_links_in_exported_pdfs]

Marked の PDF エクスポートは現在、WebKit の印刷機能を使用しています。この結果の 1 つは、目次内のリンクを含む文書内 (内部) リンクが文書内の他のポイントにジャンプしなくなることです。これは、Apple が Marked 2 が使用する WebKit のバージョンで修正するつもりはないようです。

場合によっては、スタイルが埋め込まれた HTML をエクスポートし、Web ブラウザを使用して PDF に印刷すると、良好な結果が得られることがあります。 Marked のエクスポート機能をすべて利用できるわけではありませんが、通常は内部リンクが機能する PDF を取得できます。今のところトレードオフです。

## インクルードされたファイル内の相対パス [relative_paths_in_include_files]

Marked の [include syntax][include] および [Scrivener files][scriv] を使用してインクルードされたファイルは、相対パスを使用して他のファイルを参照できます。 (_**注意:** これは、IA Writer の `/file` 構文を使用して組み込まれたファイルや CSV ファイルには適用されません_)。最近のバージョン (2.5.10 以降) では、これらのパスはベース ファイルではなく、含まれているファイルに対する相対パスです。

次のようなファイル/フォルダー構造があるとします。

    - ベースファイル.md
    - サブフォルダー
        - 付属ファイル.md
    - 画像
        - 画像1.jpg

`included_file.md` が相対パス経由で image1.jpg を参照する場合、`../images/image1.jpg`, _not_ `images/image1.jpg` として記述する必要があります。 (`..`は親ディレクトリを示します)。

[include]: http://marked2app.com/help/Multi-File_Documents.html
[scriv]: http://marked2app.com/help/Scrivener_Support.html
