# <%= @title %>

マークすると、**ライブ プレビュー**から HTML がエクスポートされます --- 画面に表示されるのと同じレンダリングされた出力です。スニペットをブログや CMS に貼り付ける必要がある場合、または任意のブラウザやホストで開くことができる、スタイルや画像が埋め込まれた自己完結型の `.html` ファイルが必要な場合は、HTML エクスポートを使用します。

一般的なワークフローは、**最初にプレビュー、2 番目に HTML のエクスポート**です。Marked でドキュメントを開くかコンパイルし、テーマを選択し、ライブ プレビューで校正し、マークアップが正しく表示されたらエクスポートします。

## HTML を取得する 2 つの方法

### HTML をコピー (スニペット)

**HTML をコピー** では、プレビューの HTML ソースがクリップボードに保存されます。これにより、WordPress、Ghost、Squarespace、フォーラム、電子メール テンプレート、または HTML フラグメントを受け入れる任意のアプリに貼り付けることができます。

* 歯車メニュー → **HTML をコピー**、またはプレビューをフォーカスした状態で {% kbd shift cmd C %}
* **レンダリングされた本文 HTML** をコピーします (`<html>` ラッパーを含む完全なドキュメントではありません)
* オプション: {% prefspane Export %} で **HTML をコピーするときに画像を埋め込む** を有効にして、貼り付けたソース内のローカル画像を `data:` URL として Base64 エンコードします。

HTML のコピーは、コピー先にすでに独自のスタイルシートがあり、コンテンツ マークアップのみが必要な場合に最適です。

### HTML (ファイル) を保存する

**HTML を保存** すると、完全な `.html` ファイルがディスクに書き込まれます。

* [エクスポートパネル](Exporting.html#drawer) ({% kbd shift cmd e %}) からエクスポート → **HTML を保存**、{% kbd cmd S %}、または **HTML**
* 保存ダイアログでファイル名と場所を選択します
* ダイアログ アクセサリでエクスポート オプションを構成します (下記を参照)

HTML の保存は、アーカイブ、スタンドアロン ファイルの共有、または結果をブラウザで直接開く場合に最適です。

## HTML オプションの保存

[HTML を保存] ダイアログには、エクスポート プロファイル ピッカーと次のオプションが含まれています。

![HTML 保存オプション][savehtml]

**出力にスタイルを含める**

チェックすると、選択したプレビュー テーマの CSS がエクスポート ファイル内の `<style>` ブロックに埋め込まれます。組み込みのテーマを選択するか、チェックボックスの横にあるスタイルメニューから [カスタムスタイル](Custom_Styles.html) を選択します。出力は、プレビューしたものと一致する、コンテンツの周囲に `<!DOCTYPE html>`、`<head>`、および `#wrapper` div を含む完全な HTML ドキュメントです。

チェックマークを外すと、レンダリングされたコンテンツのみを含む最小限の HTML ドキュメントが保存されます (マークされたテーマ CSS はありません)。独自のスタイルを提供する別のシステムに生の HTML を貼り付けたりインポートしたりする場合にこれを使用します。

**スタンドアロン HTML にローカル画像を埋め込む**

**出力にスタイルを含める** が有効な場合、ローカル画像を Base64 `data:` URL として HTML ファイル内に埋め込むこともできます。その結果、別の `images/` フォルダーを作成せずに、電子メールで送信したり、アップロードしたり、ホストしたりできる単一のファイルが作成されます。

* ローカル ドライブ上の **相対パスまたは絶対パス** によって参照されるイメージを処理します
* `file:///` URL は避けてください --- URL は確実に埋め込むことができません
* リモート画像 (http/https) は、最初にダウンロードしない限り、外部 URL のままになります。
* Base64 埋め込みでは大きなファイルが生成される可能性があります。ファイルサイズよりも移植性が重要な場合に使用してください

**構文強調表示 JavaScript を含める**

{% prefspane Preview %} で構文の強調表示が有効になっている場合、このオプションにより CDN からの highlight.js CSS と JavaScript が追加されるため、エクスポートされたファイルでコード ブロックの色が維持されます。エクスポートされた HTML を読み込むには、CDN リソース用のインターネット接続が必要です。

**MathJax または KaTeX CDN リンクを含めます**

[MathJax](MathJax.html) または KaTeX がプレビューで有効になっている場合、保存された HTML に一致する CDN スクリプトを含めることができるため、数式がブラウザーでレンダリングされます。構文の強調表示と同様に、スクリプトを自分でホストしない限り、ファイルを表示するときにネットワーク アクセスが必要です。

**CriticMarkup エクスポート タイプ**

[CriticMarkup](CriticMarkup.html) を含むドキュメントでは、エクスポートに編集されたテキスト、元のテキスト、または完全なマークアップのいずれを表示するかを選択できます。

**プロファイルのエクスポート**

保存した [エクスポート プロファイル](Exporting.html#export-profiles) を選択すると、お好みの HTML エクスポート設定 (埋め込みスタイル、画像、構文の強調表示、数式) をワンステップで復元できます。

## 組み込みテーマとカスタム テーマを使用したスタイル設定

**プレビュー スタイル** は、**出力にスタイルを含める** がチェックされている場合に HTML の外観を制御します。

1. プレビュー ウィンドウのスタイル メニューからスタイルを選択します (または、{% prefspane Style %} でデフォルトを設定します)。
2. ライブ プレビューでタイポグラフィ、見出し、コード ブロック、画像を確認します。
3. エクスポートダイアログで選択したのと同じスタイルで HTML を保存します。

すべての組み込みの Marked テーマ (Swiss、GitHub、Manuscript、その他) を埋め込むことができます。 [カスタム スタイル](Custom_Styles.html) と [スタイル マネージャー](Custom_Styles.html) のスタイルは同じように機能します。

I> 一部のプレビュー専用 CSS (固定位置、ビューポート トリック、ダーク モード `@media screen` 反転) は、Marked の外側で 1 対 1 に変換されない場合があります。保存したファイルをブラウザで開き、公開する前に確認します。

オーサリングのガイダンスについては、「カスタム CSS の作成」(Writing_Custom_CSS.html) を参照してください。

## メタデータと MultiMarkdown ヘッダー

ソース ファイルの先頭にある MultiMarkdown メタデータは、HTML エクスポートに影響を与える可能性があります。

* **`Title:`** --- 完全な HTML ドキュメントを保存するときに `<title>` 要素に使用されます
* **`XHTML Header:`** / **`HTML Header:`** --- エクスポートされた `<head>` (スクリプト、リンク タグ、メタ タグ) に追加のタグを挿入します。
* 他のメタデータ キーは [Markdown プロセッサ] に従って処理されます(Choosing_a_Processor.html)

エクスポート設定にメタデータを使用するが、キーを他の出力に表示したくない場合は、メタデータを HTML コメントで囲みます --- Marked は、ドキュメント内の任意の場所でコメントされたメタデータを検索して処理します。 「文書ごとの設定」(Per-Document_Settings.html)を参照してください。

## 複数ファイルのドキュメント

ブックや章の編集については、[複数ファイルのドキュメント](Multi-File_Documents.html) を使用します。マークすると、マージされたドキュメントがプレビューされ、コンパイルされた結果から 1 つの HTML ファイルがエクスポートされます。含まれるファイルには、ソース パスを示す HTML コメントが付けられます。どの章がどのセクションに貢献したかを監査するときに役立ちます。

## 他のアプリケーションへの貼り付け

|目的地 |推奨されるアプローチ |
| :-- | :-- |
|独自テーマのブログ・CMS | **HTML をコピー** (スニペット、埋め込まれたマーク付き CSS なし) |
|静的サイトまたはアーカイブ | **出力にスタイルを含める**を指定して HTML を保存** |
|電子メールまたはファイル共有 (添付ファイル 1 つ) | **ローカル画像を埋め込んで** HTMLを保存** |
| WordPress、Ghost、Notion など | **HTML をコピー**;エディターがローカル パスを解決しない場合は、**HTML のコピー時に画像を埋め込む** を有効にします。
|コードエディターでさらに編集する | **HTML を埋め込みスタイルなしで保存**するか、スニペットをコピーして手動で折り返す |

[リッチ テキストのコピー](Exporting.html#rtfexportoptions) (歯車メニュー) は、ターゲット アプリが HTML ソースではなくフォーマットされたテキストを受け入れる場合の代替手段です。
