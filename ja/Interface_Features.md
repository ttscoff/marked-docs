## ギアメニュー

Gear メニューには、メニューバーにあるほとんどの機能に加えて、プレビュー固有の機能がいくつか用意されています。これらの機能にアクセスするには、ウィンドウの右下の歯車をクリックするだけです。

## トップを維持する

![][5]

   [5]: 画像/KeepOnTopPin.jpg @2x width=740px height=345px class=center

左下の鍵アイコンを使用すると、プレビュー ウィンドウが最前面に表示され、他のアプリケーションに切り替えるときにプレビュー ウィンドウがそこに保持されます (フロート状態になります)。 {% prefspane General %} でウィンドウの透明度を設定すると、他のアプリケーションを使用しているときにウィンドウをフェードできるようになります。

この機能は、{% kbd shift-opt-cmd-f %} で切り替えることもできます。

## ウィンドウレベルのデフォルト

{% prefspane General %} では、[新しいウィンドウを最前面に維持] を使用して、新しいウィンドウが常に他のウィンドウの上に表示されるように設定したり、関連するドキュメントが更新されたときにウィンドウが最前面に表示されるように設定したりできます。更新時に起動するように設定された Windows は、エディターから「フォーカスを盗む」ことはなく、アクティブにならずに表示されるように起動するだけです。

## 検索

![][7]

   [7]:images/SearchBarFull.jpg @2x width=818px height=195px class=center

検索バーには {% kbd cmd F %} でアクセスでき、プレビューから単語や語句を検索できます。検索したら、{% kbd cmd G %} と {% kbd shift cmd G %} を使用して、追加の結果を前後に移動できます。

検索バーの右側にあるチェックボックスを使用すると、大文字と小文字の区別、単語全体のみ、および正規表現によって検索を絞り込むことができます。正規表現検索に加えて、ワイルドカード (*?) 検索は、regex オプションがオフの場合でも常に機能します。

検索語または語句をスラッシュで囲んで、正規表現として自動的に解釈することもできます (例: '/term.+?\b/')。


検索機能をブックマークと組み合わせて使用​​すると、見つけた場所を保存できます。 Tab ({% kbd ⇥ %}) を押してドキュメントにフォーカスし、Shift-[1-9] を押してその番号にブックマークを設定します。 (Shift キーを使用せずに) 番号を入力するだけでブックマークに戻るか、n/p を使用してドキュメントの順序でブックマークに移動することができます。 N/P は番号順に移動します。ブックマーク、目次、ミニマップ、および高速ヘッダー検索については、[ドキュメント ナビゲーション](Document_Navigation.html) を参照してください。その他のオプションについては、[プレビュー ナビゲーション](Keyboard_Shortcuts.html#previewnavigation) セクションを参照するか、「?」と入力してください。プレビューでいつでも。

> **注意:** 検索は要素間で折り返すことはできません。つまり、検索文字列は段落と見出しの間、リスト項目間などで継続することはできません。

{% kbd ctrl shift 1 %}、{% kbd 2 %}、{% kbd 3 %} を使用して、「単語全体」、「大文字と小文字を区別する」、および「正規表現」チェックボックスをそれぞれ切り替えることができます。

### 詳細検索 ###

正規表現以外の検索ではワイルドカードを使用できます。 `*` は一連のスペース以外の文字に一致し、`?` は任意の 1 文字に一致します。

`*` で検索を開始すると、jQuery セレクター検索になります。任意の jQuery セレクターを使用すると、一致するすべての要素が強調表示され、{% kbd cmd G %} と {% kbd shift cmd G %} でナビゲートできるようになります。テキスト検索の範囲を特定の要素タイプに制限するには、セレクター定義の後に二重引用符で囲んだ検索語を追加します。

    *h2「アリス」

これは `*h2:contains(Alice)` に相当します

## ドキュメント ナビゲーション (目次、ブックマーク、ミニマップ)

[ドキュメント ナビゲーション](Document_Navigation.html) ページには、目次 ({% kbd Space %} でフィルターを開くことを含む)、{% kbd f %} での高速検索、ブックマーク、およびミニマップが含まれます。

## キーボード ナビゲーション

キーボード ショートカットを使用して、プレビュー ウィンドウをすばやく移動できます。 {% kbd j %} と {% kbd k %} を使用して上下に移動し、Shift ({% kbd J %}/{% kbd K %}) を押したままにするとより速く移動します。 {% kbd t %} と {% kbd b %} はドキュメントの上部と下部に移動します (Vim ファンの場合は、{% kbd gg %} と {% kbd G %} も同様です)。 {% kbd u %} と {% kbd d %} は 1/2 ページ上下に移動します。

### ヘッダージャンプ

カンマ ({% kbd , %}) キーとピリオド ({% kbd . %}) キーを押すと、文書内のヘッダー間を前後にジャンプします。 Shift ({% kbd shift  %}) を押したままにすると、レベル 1 と 2 のヘッダーの間のみにジャンプします。


## フルスクリーン

全画面モードは、「プレビュー」メニューから切り替えるか、「{% kbd ctrl cmd F %}」と入力して切り替えることができます。

## 外部リンクをクリックする

![][10]

[10]: images/linkpopover.png @2x width=706px height=213px class=center

ドキュメントのプレビューで外部リンクをクリックすると、デフォルトのブラウザでドキュメントが開きます。クリックして押したままにし、放したときに「マーク」には 3 つのオプションが表示されます。リンクの URL をクリップボードにコピーするか、リンクを検証するか、デフォルトのブラウザでリンクを開くことができます。プレビュー内の任意の場所をクリックするだけでウィンドウを閉じます。この機能は、{% prefspane Preview %} (「リンク ポップオーバーを有効にする」) で無効にすることができます。

[YouTube の概要ビデオ](https://www.youtube.com/watch?v=nrt7YZPrnv0&list=PLNoOPRHOBV0ptk_ka386xltm3VyVw6QgZ&index=1) をご覧ください。

## 折りたたみ可能な見出し ##

{% prefspane Preview %} で「見出しのセクションを折りたたむ」が有効になっている場合、見出しをクリックすると、その見出しと同じレベルの次の見出しの間のセクションが折りたたまれます。そのセクション内のサブセクションは非表示になります。必要に応じて、この動作を {% kbd cmd %} クリックに制限できます。

![][13]

[13]: images/headlines.jpg @2x width=740px height=287px class=center

{% kbd opt  %} を押したままクリックすると (または {% kbd cmd %} を押しながらクリックすると)、クリックした見出しの下にあるすべての子セクションが展開または折りたたまれます。 {% kbd shift  %} (Shift) キーを押したままクリックすると、同じレベルにある他のすべてのセクションが折りたたまれ、クリックしたセクションのみが表示されます。

折りたたまれたセクションはタイトルの右側に黄色のハイライトでマークされ、項目をクリックしたときに何が起こるかをカーソルが示します。

表示する必要がある編集が行われた場合、または現在折りたたまれたセクション内にある目次セクションをクリックした場合、必要な親セクションが展開されて表示されます。

「すべてのセクションを折りたたむ」（{% kbd opt cmd left  %}）および「すべてのセクションを展開する」（{% kbd opt cmd right  %}）を使用すると、文書内のすべてのセクションを一度に折りたたんだり展開したりできます。

詳細については、[YouTube のドキュメント ナビゲーション ビデオ](https://www.youtube.com/watch?v=Z2p5BvBpbmA&list=PLNoOPRHOBV0ptk_ka386xltm3VyVw6QgZ&index=2) を参照してください。

## スクロールして編集

Marked の「編集までスクロール」機能は、最新の更新と最後の更新の違いを追跡し、最後に変更を加えたポイントを見つけようとします。 「マーク付き」は常にこれを追跡し、最初に検出された変更の位置を示す小さな赤い線がプレビューに表示されます。 {% prefspane Preview %} で、「最初の編集までスクロール」をオンにすると、プレビューが更新されると、ビューがその位置までゆっくりスクロールされます。

「最初の編集までスクロール」をオフにしても、プレビュー内でいつでも「e」キーを押して、最後に保存された編集ポイントに移動できます。
