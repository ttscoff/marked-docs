# <%= @title %>

リンク検証では、URL の宛先に ping を実行し、エラーをテストします。これは、公開ドキュメント内の壊れたリンクや無効なリンクを回避するのに役立ち、特にブロガーにとって便利です。

## 単一リンクの検証

![][1]

   [1]:images/validate_single.png @2x width=832px height=267px class=center

プレビュー内のリンクをクリックして点滅するまで押し続け、放すとリンク アクション メニューが開きます。 「リンクの検証」を選択してテストを実行します。結果がポップアップに表示されます。

## すべてのリンクを検証する

![][2]

   [2]: 画像/スクリーンショット/mainwindow-feature-urlvalidate-crop.jpg @2x width=1089px height=300px class=center

歯車メニューまたは右クリックメニューから「すべてのリンクを検証」(ショートカット {% kbd ctrl cmd L %}) を選択します。ドキュメント内のすべてのリモート リンクがチェックされ、結果がポップアップに表示されます。ポップアップ内のリンクをクリックすると、文書内のそれぞれのリンクまでスクロールして強調表示されます。

有効な URL は、ポップアップの上部にある [有効な URL を非表示] ボタンを使用して非表示にすることができます。これにより、エラーステータスを返した URL のみが表示されます。

Esc キーを押すと、検証結果が非表示になります。 {% kbd ctrl cmd L %} または歯車メニューを使用して再度表示できます。

## 自動的に検証する

プレビュー設定 (またはリンク検証ポップアップの下部) で「更新時に URL を自動的に検証する」をオンにします。ドキュメントが読み込まれると、含まれているリンクがバックグラウンドでテストされます。エラーがある場合にのみダイアログが表示されます。

ポップアップを無効にするには、設定でポップアップをオフにするか、ポップアップ ウィンドウの下部にあるボックスのチェックを外します。
