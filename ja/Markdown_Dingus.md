## Dingus での編集

Markdown 入力ペインには、スマートな編集機能が含まれています。

### スマート改行 (リターンキー)

Return キーを押すと現在の行に適応します。

* __Lists__: リスト行 (`-`、`*`、`+`、または `1.`)、
  正しいマーカーを持つ新しいリスト項目を挿入します。番号付き
  リストは次の番号に続きます。
* __Blockquotes__: `>` で始まる行に、

  そして柵を閉める。
* __その他の行__: 通常の改行を挿入します。

## ディンガスを使用する理由

### プロセッサーの違いを理解する

Markdown プロセッサが異なれば、構文の処理方法も異なります。の
ディンガスは次のことに役立ちます。

* __出力の比較__: 各プロセッサーがどのようにレンダリングするかを正確に確認します
  同じマークダウン
* __デバッグの問題__: 特定の構文が機能しない理由を特定する
  期待どおりに動作する
* __構文を学ぶ__: 微妙な違いを理解する
  プロセッサ実装間

### 書く前のテスト

特定の Markdown スタイルにコミットする前に、
書類:

* __Validate Syntax__: マークダウンがレンダリングされることを確認します。
  正しく
* __プロセッサの選択__: どのプロセッサが最適に動作するかを決定します
  あなたのコンテンツのために
* __安全に実験__: 影響を与えずに新しい構文を試します
  実際の書類
