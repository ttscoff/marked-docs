# マークダウンファイアー

Markdownifier は、Web ページからコンテンツを自動的に抽出し、クリーンな Markdown 形式に変換するツールです。 Web コンテンツをインテリジェントに処理して、広告、ナビゲーション要素、その他の煩雑な要素を除外して、意味のあるテキストと構造だけを提供します。

## 仕組み

Markdownifier は、高度なコンテンツ抽出アルゴリズムを使用して次のことを行います。

1. **Web ページのコンテンツを取得して分析**
2. **主要な記事を特定する** テキストと構造
3. コンテンツを**クリーンアップしてフォーマット**し、適切な Markdown に変換します
4. **広告、ナビゲーション、その他の非コンテンツ要素を除外**
5. ヘッダー、リスト、リンクなどの重要な書式設定を **保持**

## Markdownifier を開く

Markdownifier にアクセスするには、{% appmenu File, New, Markdownify URL (@~k) %} を開きます。 Markdownify する URL を入力し、Return ({% kbd return %}) を押します。

## Markdownifier の使用

### 基本的な使い方

1. 上記のいずれかの方法を使用して **Markdownifier を開きます**
2. **テキストフィールドにURLを入力**
3. **[自動]** をクリックするか、`Return` を押してコンテンツを抽出します
4. 抽出されたコンテンツは、新しいマーク付きドキュメントで **自動的に開きます**

### マニュアルコンテンツの選択

自動抽出で必要なコンテンツがキャプチャされない場合:

1. **「手動」** ボタンをクリックして、Web ビューにページをロードします
2. **ナビゲートしてスクロール**して、必要なコンテンツを見つけます
3. Web ページ上に表示される **[コンテンツの抽出]** ボタンをクリックします。
4. 選択したコンテンツは Markdown に変換され、Marked で開きます。

### クリップボードの統合

Markdownifier は、開いたときにクリップボード内の URL を自動的に検出します。

- URL が見つかった場合は、URL フィールドに **事前入力**されます。
- 処理するには、**「自動」** をクリックするか、`Return` を押す必要があります。
- これにより、クリップボード URL の誤った処理が防止されます。

## コンテンツ処理

### コンテンツの自動検証

Markdownifier は、抽出されたコンテンツをインテリジェントに検証して、意味のあるテキストが含まれていることを確認します。

- **メタデータを除去** (YAML フロントマター、MultiMarkdown ヘッダー)
- **リンク定義を削除** および参照形式のリンク
- **スタンドアロン URL とナビゲーション要素を除外**
- **空白を圧縮**して正確な長さを評価します
- **実際のコンテンツには少なくとも 200 文字が必要です**

抽出されたコンテンツが短すぎる場合、またはほとんどがナビゲーション/広告であるように見える場合、Markdownifier は自動的に手動選択モードに戻ります。

### コンテンツのフォーマット

抽出されたコンテンツは、次のようなクリーンな Markdown としてフォーマットされます。

- **ソースリンク** 上部: `[source](original-url)`
- **H1 タイトル挿入** (必要な場合)
- **保存されたリスト** (順序付きおよび順序なし)
- **維持されたリンク**と強調の書式設定
- **適切な間隔で段落を整理**する

## 安全機能

### 衝突防止

Markdownifier には、クラッシュを防ぐためのいくつかの安全対策が含まれています。

- **問題のある URL をブロック** (広告ネットワーク、追跡サービス、暗号関連コンテンツ)
- **レンダリングの問題を引き起こす可能性のある破損した画像をフィルタリング**
- **不安定性を引き起こす可能性がある高度な Web 機能を無効にします**
- **自動クラッシュ回復** (セーフ モード フォールバック付き)

### プライバシー保護

- **プライベート ブラウジング モード** により、追跡と Cookie が防止されます
- **セキュリティのためプラグインや Java** は実行されません
- **制限された JavaScript** と暗号 API ブロック
- **リソース フィルタリング** により、追跡と広告コンテンツがブロックされます

## トラブルシューティング

### コンテンツが抽出されませんでした

自動抽出が失敗した場合:

1. [手動] ボタンを使用して **手動選択を試してください**
2. **サイトに JavaScript が必要かどうかを確認します** - 一部のサイトでは手動でロードする必要があります
3. **URL がアクセス可能であり、記事のコンテンツが含まれていることを確認します**
4. アクセスをブロックする可能性のある **ペイウォールまたはログイン要件を探します**

### WebView の問題

Web ビューが不安定になった場合:

1. Markdownifier は **自動的にセーフ モードに入ります**
2. クラッシュを防ぐため、**JavaScript が無効になります**
3. **手動選択の代わりに「変換」ボタンを使用します**
4. **Markdownifier を閉じて再度開き**、リセットします

### コンテンツがありません

重要なコンテンツが抽出に含まれていない場合:

1. **自動アルゴリズム**により除外された可能性があります
2. **手動選択を使用**して、必要な特定のコンテンツを選択します
3. **ソース HTML をチェック**して、コンテンツが動的に読み込まれているかどうかを確認します
4. サイトの構造が複雑な場合は、**別の URL を試してください**

## 最良の結果を得るためのヒント

### URL の選択

- **ホームページやカテゴリ ページではなく、記事の URL を使用します**
- **トラッキング パラメータを含む URL は可能な限り避けてください**

### コンテンツの品質

- **長い記事**は、一般的に短い投稿よりも適切に抽出されます
- **適切な見出しを備えた適切に構造化されたコンテンツ**が最も効果的です
- **自動抽出のために重い JavaScript を使用するサイトを避ける**

### 手動選択

- **ページが完全にロードされるまで待ってから**抽出してください
- **コンテンツをスクロールして**、すべてがロードされていることを確認します
- **領域の上にマウスを移動**して、抽出するすべてのコンテンツを含む最小の青いボックスを選択します
- 必要なコンテンツが見つかったら、**クリック**

## 高度な機能

### バッチ処理

Markdownifier が一度に 1 つの URL を処理している間、次のことができます。

- **Markdownifier を複数回開くことで、複数の URL をキューに入れる**
- **サービス統合を使用**して、他のアプリケーションからの URL を処理します
- **抽出したコンテンツをコピー**し、既存のマークされたドキュメントに貼り付けます

### Marked との統合

抽出されたコンテンツは次のマークで開きます:

- **記事タイトルに基づいた自動ファイル名**
- ドキュメントメタデータ内の **ソース URL の保存**
- **読み取りとエクスポートの完全なマーク付き機能**)

## 技術的な詳細

### サポートされているコンテンツ タイプ

- 標準マークアップを含む **HTML 記事**
- **ブログ投稿**およびニュース記事
- **ドキュメント**とヘルプ ページ
- **フォーラムの投稿**とディスカッションのコンテンツ

### 制限事項

- **ペイウォールサイト**ではログインと手動抽出が必要な場合があります
- **JavaScript を多用するサイト** では手動での選択が必要な場合があります
- ページの読み込み後に読み込まれた **動的コンテンツ** は失われる可能性がありますが、手動で抽出するとキャプチャできます
- **複雑なレイアウト**には、不要なナビゲーション要素が含まれる可能性があります

Markdownifier は、複雑な Web サイトや問題のある Web サイトにフォールバック オプションを提供しながら、Web コンテンツの抽出を可能な限りシンプルかつ信頼性の高いものにするように設計されています。
