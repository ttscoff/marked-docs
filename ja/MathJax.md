数字は言葉と同じくらい重要です。

## MathJax で数式をプレビューする

![][1]

   [1]:images/mathjax.jpg @2x width=713px height=512px class=center

{% prefspane Style %} で MathJax をオンにすると、必要なスクリプトと CSS がプレビューに含まれます。その後、Markdown ドキュメントで MultiMarkdown 数学構文を使用できるようになり、結果が表示されます。

MMD MathJax 構文の例:

    \\\\[ {e}^{i\pi } 1=0 \\\\]

エクスポートされた HTML ファイルに MathJax を含めることを選択した場合、埋め込まれた MathJax コードの代わりに CDN リンクが使用されます。これには、レンダリングされた MathML を表示するためにインターネット接続が必要です。

## MathJax ソース: ローカル vs CDN

MathJax が有効になっている場合、Marked は次のいずれかからロードできます。

- **ローカル**: アプリ内にバンドルされている MathJax v3 のコピー (読み込みが速く、オフラインで動作します)。
- **CDN**: 公式 MathJax v3 CDN (常に最新、アプリ バンドルの影響なし)。

{% prefspane Style %} の **MathJax Source** ポップアップでは、以下を選択できます。

- **ローカル** – アプリバンドルの ES5 `tex-chtml.js` コンポーネントを使用します。
- **CDN** – 同じコンポーネントを CDN からロードします。これにはインターネット接続が必要です。

エクスポートされた HTML ファイルは、プレビュー ソースに関係なく、常に CDN から MathJax を参照するため、自己完結型で小さいままになります。

## 方程式の番号付け

{% prefspane Style %} で方程式の番号付けを有効にできます。これは MathJax と KaTeX の両方で動作しますが、内部的には異なる方法で実装されます。 MathJax v3 の場合、Marked は次のように設定を適切な MathJax 構成にマップします。

- 「数値方程式」は、数値を表示するかどうかを制御します。
- 「サイド」オプションは、数字を左側に表示するか右側に表示するかを制御します。
- 「AMS のみ」オプションは、番号付けを AMS スタイルの方程式に制限します。

これらのオプションは、内部では MathJax の `tex.tags` および `tex.tagSide` 設定に対応しています。

## 追加パッケージ

MathJax v3 はモジュール式です。マークを付けると、コア TeX/AMS パッケージが常に有効になります。オプションで、{% prefspane Style %} で追加のパッケージを有効にすることができます。

- **物理** (`physics`) – 物理表記と利便性。
- **化学** (`mhchem`) – 化学方程式。
- **ブラケット** (`braket`) – ディラックのブラケット表記。
- **太字記号** (`boldsymbol`) – 太字の数学記号。

**追加パッケージ…** をクリックすると、これらのパッケージをオンまたはオフにできる小さなチェックリストが開きます。変更は、次回マークがプレビューで数式をレンダリングするときに有効になります。

## MathJax の詳細設定

[**詳細構成**] フィールドに有効な JSON オブジェクトを追加することで、Marked のデフォルトに加えて追加のカスタム構成を適用できます。このフィールドは、MathJax がロードされる前に、MathJax v3 構成オブジェクト (`window.MathJax`) にマージされます。 [MathJax v3 でサポートされている任意のオプション](https://docs.mathjax.org/en/latest/options/) を含めることができます。例:

    {
        "テックス": {
            "inlineMath": [["$","$"],["\\\\(","\\\\)"]],
            "displayMath": [["$$","$$"],["\\\\[","\\\\]"]],
            「マクロ」: {
                "tr": "{\\\\scriptscriptstyle\\\\mathrm{T}}"
            }、
            "パッケージ": { "[+]": ["物理"] }
        }
    }

この例では、TeX 区切り文字を調整し、`\tr` マクロを追加し、デフォルトのセットに加えて `physics` パッケージを有効にします。これらの設定を使用すると、次のすべてが適切にレンダリングされます。

    括弧を使用したインライン式、\\\\({x}^{2} {y}^{2}=1\\\\)、またはドル記号を使用した ${x}^{2} {y}^{2}=1$。

    エスケープされた括弧を使用して表示します。

    \\\\[ {e}^{i\pi } 333=0 \label{testme} \\\\]

    または二重ドル記号を使用して:

    $${x}_{1,2}=\\frac{-b\pm \sqrt{{b}^{2}-4ac}}{2a}$$

![][2]

   [2]:images/mathjax2.jpg @2x width=837px height=260px class=center

追加の構成は既存のオブジェクトを拡張するため、指定されたプロパティのみがオーバーライドされます。指定されていないオプションは、現在のプリセットのデフォルトのままになります。

標準以外の区切り文字を使用して MultiMarkdown プロセッサを使用すると、式内の文字が解析されるため、`*` や `^` などの記号は、MathJax プロセッサを破壊する文字体裁の変更を引き起こすことに注意してください。このような場合の最善の解決策は、[プロセッサ設定](x-marked-3://pref/processor) で割引プロセッサを使用することです。

MathJax または KaTeX が有効な場合、Marked はちょっとした魔法を実行し、現在のプロセッサ (MultiMarkdown またはdiscount) と可能な限り互換性があるように数学構文を変換します。これはどのような状況でも優れているはずですが、問題が発生している場合は、[サポートに連絡](https://support.markedapp.com/questions/add)してください。


## カテックス

[katex]: https://katex.org/

MathJax の代替として [KaTeX][] が利用可能です。軽量なので読み込みが速く、多数の数式を含むドキュメントに最適です。ただし、それほど多くの機能はなく、MathJax (または LaTeX) で機能する一部の方程式はサポートされていない可能性があります。

## 式の自動番号付け [番号付け]

{% prefspane Style %} で方程式の番号付けを有効にできます。これは MathJax と KaTeX の両方で機能します。数値を方程式の左側に表示するか右側に表示するかを選択できます。

### MathJax の場合

MathJax では、これは次の設定を使用します。

    {
      TeX: {quationNumbers: {autoNumber: "all" } }
    }

AMS 方程式に番号のみを付けたい場合は、「サイド」ドロップダウン メニューの右側にある「AMS のみ」を選択します。

### KaTeX で

KaTeX は方程式の番号付けを提供しません。これを Marked でシミュレートするには、CSS が使用され、すべての表示方程式に番号が付けられます。

## エクスポートの問題

リッチ テキスト形式では、方程式 (MathJax または KaTeX による) は処理されません。特殊なフォントを含めようとすると、想像以上に混乱が生じるため、出力ドキュメントでは非表示になります...これは、いつか修正したいと思っていますが、現時点では欠点です。
