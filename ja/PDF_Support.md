# <%= @title %>

マークを付けると、PDF ドキュメント (`.pdf`) を直接開くことができます。ファイルを [マーク] にドラッグするか、{% appmenu File, Open... ({{cmd}}O) %} を使用します。ドキュメントはプレビューとエクスポートのために Markdown に変換されます。

PDF インポートは、**小さいテキストベースの PDF** (スライド、記事、短いレポート) で最も効果的に機能します。大きなマニュアル、書籍、スキャンされたドキュメントはサポートされていますが、多くの場合、変換に時間がかかるか不完全になります。以下の **制限事項** を参照してください。

Marked はまだ **プレビュー** ツールです。Marked 内の PDF を編集することはできません。 {% kbd cmd E %} を使用して PDF を **プレビュー** (またはシステムのデフォルト) で開き、ディスク上のファイルが変更されるとマークが更新されます。

## 変換の仕組み

PDF インポートでは、[pdf22md](https://github.com/twardoch/pdf22md) ライブラリ (MIT ライセンス。[pdf22md ライセンス](PDF22MD_License.html) を参照) を使用します。マークは、**変換中**という短い通知を表示しながら、バックグラウンドで変換を実行します。

コンバーター:

- PDFKit を使用してデジタル PDF から **テキスト**を抽出します
- 埋め込みテキストが欠落しているページで **Vision OCR** を使用します (スキャンで一般的)
- 可能な場合はフォント サイズから **見出し** を検出します
- **画像**を、生成されたマークダウンの隣の `assets` フォルダーに保存します

マークは、アプリでの pdf22md のオプションの AI クリーンアップを有効にしません**。変換の品質は、PDF の作成方法によって異なります。

## キャッシュとライブ プレビュー

変換された Markdown と画像は Marked の Watchers キャッシュ (`~/Library/Caches/Marked/Watchers/PDF/`) に保存されます。元の PDF パスは、ファイル監視用のドキュメント ソースのままです。

別のアプリケーションで PDF を保存または置き換えると、Marked は変更を検出し、自動的に再変換します ([RTF](RTF_Support.html) および [Scrivener](Scrivener_Support.html) と同じ合体リロード動作)。

## エクスポートとその他の機能

変換後、Marked はドキュメントを他のコンパイル済みソースと同様に扱います。エクスポート、統計、およびほとんどのプレビュー機能は、生成された Markdown に対して実行されます。エクスポートするまで、プレビュー内の画像パスはキャッシュされたアセットを指します。

## 制限事項

それに応じて期待を設定してください。PDF-to-Markdown は便利ですが、完璧ではありません。

**うまく機能するもの**

- 実際のテキストが埋め込まれた **ベクターおよびテキストベースの PDF** (Word、Pages、InDesign などからエクスポート)
- **適度な長さ** - 通常、数十ページが適切な構造で適切な時間内に変換されます。

**信頼できないもの**

- **OCR (スキャンされた PDF)** — Vision は欠落しているテキストを補完しますが、専用の OCR ツールと比較すると精度が劣ることがよくあります。タイプミス、壊れた単語、欠落した列が予想されます
- **表** — レイアウトはテキストの位置から推測されます。マージされたセル、ネストされたテーブル、複雑なグリッドがクリーンな Markdown テーブルとして存続することはほとんどありません。
- **画像の配置** — 図は `assets/` に抽出されますが、画像の配置、キャプション、テキストの回り込みは確実に保持されません。

**サイズと性能**

- **大きな PDF** (ユーザー ガイド、教科書、長いマニュアル) は、変換に **非常に長い時間**がかかったり、大量のメモリを使用したり、有用な Markdown を生成できなかったりする可能性があります。変換がバックグラウンドで実行されている間、Marked は応答性を維持しますが、500 ページのマニュアルが正常に完了するという保証はありません
- コンテンツがほとんどまたはまったくない状態で変換が完了した場合、空白のプレビューを残すのではなく、マーク付きでエラーが表示されます。

**その他の制限**

- **パスワードで保護された PDF** は v1 ではサポートされていません
- **Markdown に埋め込まれた PDF 画像** (`![]()` は `.pdf` ファイルを参照) は PDF インポートとは無関係です。メインドキュメントとして `.pdf` を開くだけで変換が開始されます。

Word 文書の場合は、[DOCX の操作](Working_with_DOCX.html) を使用します。リッチテキストの場合は、[RTF および RTFD のサポート](RTF_Support.html) を使用します。
