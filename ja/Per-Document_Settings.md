# <%= @title %>

Marked を使用すると、ドキュメントの特定の属性を MultiMarkdown メタデータ形式で設定できるようになります (詳細は後述)。これらを使用すると、デフォルト設定を変更せずに、そのドキュメントにのみ影響する特性とスタイルを定義できます。

ほとんどの MultiMarkdown ヘッダーはプレビューでは無視されますが、次のヘッダーは許可されており、レンダリングに影響します。最終出力にレンダリングされる他のメタデータを含めることができます。Marked は、以下にリストされていないキーを無視するだけです。 HTML として保存し、テンプレートを *含めない場合、Marked はすべてのメタデータ キーを期待どおりにレンダリングします。

## メタデータ形式

メタデータは、Markdown ファイルの先頭、または YAML ヘッダーの直後に入力されます。これらは、キー、その後に続くコロン、オプションのスペースまたはタブ、および値で構成されます。

	メタデータの例: キーの例

複数のメタデータ エントリはそれぞれ独立した行に配置する必要がありますが、間に改行は入れません。最後のメタデータ エントリの後には、ドキュメント テキストの開始前に空白行が続く必要があります。

	最初: これは最初のメタデータ エントリです
	2 番目: これは 2 番目のエントリです
	3 番目: 最後のメタデータ エントリ

	# 文書本文の先頭

## マークされたメタデータ キー

### 他のプロセッサのメタデータを非表示にする [hidingmeta]

**注意:** カスタム プロセッサを使用している場合、またはこのメタデータを処理しないソースに Markdown を直接投稿している場合でも、メタデータの前後に HTML コメント マーカーを追加することで使用できます。 MultiMarkdown や他のプロセッサとは異なり、Marked はドキュメント内の任意の場所でこれらのタグを見つけて処理し、出力からそれらを削除します。したがって、ヘッダー内の次のコードは、Marked で必要な結果を提供しますが、他の場所では表示されません。

	<!--
	マークされたスタイル: 私のカスタム スタイル
	カスタムプロセッサ: true
	-->

*メタデータ キーがスペースやタブを含まない行の先頭で始まることを確認し、値の後の行には何も置かないでください。*

### ドキュメントごとのスタイル

「Marked Style:」キーは、ドキュメントのプレビュー スタイルを設定します。値は、デフォルトのスタイルの名前、または設定で定義した [カスタム スタイル](Custom_Styles.html) の名前またはパスにすることができます。このキーが見つかり、Marked が認識しているスタイルと一致する場合、そのスタイルは、そのキーを含むドキュメントがロードされるたびにプレビューに使用されます。

**例**

	マークされたスタイル: 誠実な市民

### 引用言語

デフォルトでは、Marked は英語形式の引用符を使用します。 「Quotes Language:」キーを使用して、ドキュメントごとにこれを変更できます。利用可能な言語は次のとおりです。

* オランダ語
* 英語
* フランス語
*ドイツ語
*ギルメット
* スウェーデン語

**例**

	引用言語: フランス語

	フランス語の「引用符」を作成します。

### 基本ヘッダーレベル

「Base Header Level:」キーを使用して、Marked がカウントを開始するヘッダー レベルを設定できます。これは 1 ～ 6 の数字である必要があり、「#」ヘッダーのレンダリング方法が変更されます。ヘッダー レベルを 3 に設定すると、通常は第 1 レベルのヘッダー (h1) が第 3 レベルのヘッダー (h3) としてレンダリングされ、階層内の後続のヘッダーは 2 つ上にシフトされます。

**例**

	ベースヘッダーレベル: 3

	# この見出しは h3 として表示されます

	## この見出しは h4 になります

	次のようにレンダリングされます。

	<h3>この見出しは h3 として表示されます</h3>

	<h4>この見出しは h4 になります</h4>

### カスタムプロセッサ

[カスタム プロセッサー](Custom_Processor.html#preprocessor) で詳しく説明されているように、メタデータを使用してカスタム プロセッサーとカスタム プリプロセッサーを有効または無効にすることができます。

    カスタムプロセッサ: true
    カスタム プリプロセッサ: false

「true」または「false」は、プリプロセッサ/プロセッサをオンまたはオフにします。

「プロセッサ」値を「multimarkdown」または「discount」に設定すると、内部プロセッサのいずれかが使用されるように強制できます。このドキュメントごとの設定は、{% prefspane Processor %} のデフォルト設定を変更しません。

### ヘッダー/フッターを印刷する

次のキーを使用して、{% prefspane Export %} の印刷ヘッダーとフッターの設定を上書きできます。

	ヘッダーを左に印刷します:
	印刷ヘッダーの中心:
	ヘッダーを右に印刷:
	フッターを左に印刷:
	フッターの中央を印刷:
	フッターを右に印刷します:

これらには、`%title`、`%page`、`%total` などの [印刷変数](Exporting.html#headersandfooters) や、`%md_[key without spaces]` を使用した他のメタデータへの参照が含まれます。

### 印刷余白

`Margins:` キーを使用して、印刷およびページ分割された PDF 出力のページ余白を設定します。値はポイント単位です。 `px`、`pt`、`em` などの接尾辞は無視されます。垂直方向と水平方向のマージンに 2 つの数値を指定するか、上、右、下、左に 4 つの数値を指定します。

	余白: 10 20
	余白: 10、20、10、20

メタデータのマージンは、PDF エクスポート パネルの {% prefspane Export %} 設定およびマージン フィールドをオーバーライドします。

### JavaScript の挿入

このメソッドは、ドキュメントの `<head>` タグに含まれるデータを指定します。 Marked は、ドキュメント全体の出力を除き、このキーのほとんどの値を無視しますが、この方法で含まれるスクリプトは尊重されます。ここで定義されたスクリプト タグはヘッダーには含まれませんが、終了 `</body>` タグの前に追加されます。 jQuery はすでにロードされているため、挿入するスクリプトでそれを利用できます。

**例**

	XHTML ヘッダー: <script>(function($){$('#wrapper').fadeOut();})(jQuery);</script>

		-または-

	XHTML ヘッダー: <script src="myfancyscript.js"></script>
