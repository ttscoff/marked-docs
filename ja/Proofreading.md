# <%= @title %>

歯車メニューから校正モードに入ります。これは、主に他の人からテキストを受け取り、コメントやフィードバックを提供する必要がある編集者向けに設計された実験的な機能です。

校正モードでは文書の更新を停止し、文書が更新されたときに注釈やメモが失われるのを防ぎます。上部バーに赤いインジケーターが表示され、校正モードがアクティブであることが示されます。

キーボード ナビゲーション、ブックマーク、キーワード ハイライトは校正時にすべて機能します。

## 注釈

校正モードで文書内のテキストを選択すると、いくつかの異なるハイライト タイプから選択できるポップアップが生成されます。テキストに追加するハイライトの種類をクリックするか、「キャンセル」ボタンをクリックするか、ポップアップの外側をクリックしてキャンセルします。

ハイライトを追加したら、ハイライトをクリックして短いメモを追加できます。ポップアップには、強調表示されたテキストに関連するメモを入力できるテキスト フィールドが含まれます。すでにメモがあるハイライトをクリックすると、メモを削除および編集できます。

メモは HTML に保存するときに**のみ**エクスポートされます。ハイライトは、RTF や PDF を含むほとんどのエクスポート形式に残ります。

## スペルチェック

校正モードでは、歯車メニュー {% appmenu {{gear}}, Proofing, Show Spelling Errors %} からシステム全体のスペル チェッカーにアクセスできます。サイズの大きなドキュメントでは時間がかかり、プロセスに 30 秒以上かかる場合は警告が表示されます。スペル チェッカーは Marked の Web プレビューでは機能しないため、これを機能させるために「ハック」が実装されていますが、これは高速なものではありません。

スペルチェックが実行されると、「スペル推測」パネルを開いて文法チェックをオンにしたり、エラーを修正するための提案を確認したりできます。これらをその場で編集できないとマークされているため、結果を利用するには元のドキュメントに戻る必要があります。

*ショートカット:* {% kbd ctrl opt cmd S %} はスペルチェッカーを実行します。 {% kbd ctrl opt cmd N %} はドキュメント内の次のエラーに移動し、{% kbd ctrl opt cmd G %} は推測パネルを表示します。
