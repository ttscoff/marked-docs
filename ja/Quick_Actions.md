# <%= @title %>

クイック アクション パレットは、Marked の検索可能なコマンド ランチャーです。メイン メニュー バーとプレビュー **歯車メニュー**、さらにメニューに表示されないプレビュー専用のキーボード コマンド (**自動スクロール**など) からのアクションを収集します。やりたいことはわかっているが、それがどのメニューに含まれているか思い出せない場合に使用します。

## クイック アクション パレットを開く

{% kbd shift cmd P %} または {% appmenu File, Quick Actions… %} からパレットを開きます。パネルは、現在のドキュメントの上にフローティング ウィンドウとして表示されます。

同じショートカットをもう一度押すか、**Escape** を押してパレットを閉じます。パレットがすでに開いている場合は、メニューから **クイック アクション…** を選択してもパレットが閉じます。

## ショートカットのカスタマイズ

デフォルトのショートカットは {% kbd shift cmd P %} です。変更するには、{% prefspane General %} を開き、**ショートカット** セクションの **アクション パレットを開く** の下に新しい組み合わせを記録します。

**マークをアクティブにする** や **最初のウィンドウを開く** とは異なり、クイック アクション ショートカットは、マークがアクティブなアプリケーションである場合にのみ機能します。設定に合わせて {% appmenu File, Quick Actions… %} メニュー ショートカットを更新します。

## 検索とフィルター

パネルの上部にある検索フィールドに入力して、リアルタイムでコマンドをフィルタリングします。マッチングは曖昧で寛容です:

- 語順は関係ありません (`view source` は **Toggle Source View** と一致します)
- イニシャルは適切に機能します (`sv` は **ソース ビュー** と一致します)
- 折りたたまれた一致ではスペースのないコマンドが検索されます (`akdoc` は **Ask About Document** に一致します)

各結果では、左側にコマンド名が、右側にそのキーボード ショートカット (使用可能な場合) が灰色で表示されます。一部のコマンドでは、アクションがサブメニューまたはプレビュー エンジンからのものである場合、ブレッドクラム パス (例: `Preview › Autoscroll`) も表示されます。

## リストに表示される内容

パレットには次のものが含まれます。

- メイン メニュー バーの **メニュー コマンド** (**スタイル**、**履歴**、**ウィンドウ** タブを開くなどの動的メニューを含む)
- プレビュー ウィンドウからの **歯車メニュー** コマンド
- プレビュー内のキーボード マップからの **プレビュー ショートカット** (ナビゲーション、自動スクロール、ブックマーク、検索、その他のプレビュー専用アクションを含む) (プレビュー ヘルプ HUD に表示されるのと同じコマンド)

同じコマンドが複数の場所に出現する場合、「マーク」は最短のメニュー パスを表示し、重複したショートカット情報をマージします。

## キーボード ナビゲーション

クイック アクション パレット全体をキーボードから操作します。

- **↑/↓ 矢印キー**: フィルタリングされたリスト内を移動します。
- **Return**: 選択したコマンドを実行します
- **エスケープ**: パレットを閉じます
- **⌘ キーのショートカット**: パレットを閉じて、対応するメニュー コマンドを実行します (たとえば、リストで選択する代わりに、{% kbd cmd U %} を押して **ソース ビューの切り替え** を実行します)

単純な入力 (Command キーを使用しない) では、検索フィールドがフィルター処理されます。プレビュー専用の単一キー ショートカット (自動スクロールの `s` など) はリストをフィルターします。 **Return** を押して、選択したコマンドを実行します。

## コマンドの実行

メニュー コマンドを選択すると、動的メニュー項目やギア メニュー項目など、メニューから選択するのと同じ方法でそのコマンドが送出されます。

**プレビュー ショートカット**を選択すると、アクティブなプレビューで関連するアクションが実行されます (たとえば、自動スクロールの切り替えや次のヘッダーへのジャンプなど)。これらのコマンドには、プレビューのある開いたドキュメントが必要です。使用可能なプレビューがない場合でもパレットは開きますが、プレビューのみのアクションは効果がありません。

関連するファイルの切り替えについては、「クイックオープン」(Quick_Open.html)を参照してください。完全なプレビュー キーボードのリファレンスについては、[キーボード ショートカット](Keyboard_Shortcuts.html) を参照するか、プレビューで {% kbd h %} を押してヘルプ HUD を表示します。
