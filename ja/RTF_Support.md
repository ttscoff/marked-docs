# <%= @title %>

マーク付きでは、リッチ テキスト形式 (`.rtf`) および RTFD (`.rtfd`) ドキュメントを直接開くことができます。ファイルを [マーク] にドラッグするか、{% appmenu File, Open... ({{cmd}}O) %} を使用します。ドキュメントはプレビューとエクスポートのために Markdown に変換されます。

これは、**Pages**、**TextEdit**、**Word**、および RTF または RTFD を保存するその他のアプリのドキュメントでうまく機能します。マークはまだ **プレビュー** ツールです。元のアプリケーションで編集し、保存時にマークが更新されます。

## 変換の仕組み

Marked は、システム テキスト エンジンを使用して RTF を HTML に変換し、その後 Markdown に変換します。コンバーター:

- **大きな段落フォント サイズ**を見出しレベルにマッピングします (文書内の最も一般的な本文サイズと比較して)
- 可能な場合は **太字**、*斜体*、およびリンクを保持します
- RTFD バンドルおよび埋め込まれた添付ファイルから **画像** を抽出します
- RTF ファイル名を画像キャプションとして**扱いません** (下記の画像を参照)

同じ変換パイプラインが、Scrivener RTF コンパイルと他のドキュメントに含まれる RTF ファイルに使用されます。

## ライブプレビュー

RTF または RTFD ファイルを別のアプリケーションに保存すると、Marked は変更を検出し、プレビューを自動的に更新します。

## 画像

RTF は、Cocoa が HTML にエクスポートするような個別の「キャプション」フィールドを定義しません。レイアウト内でキャプションのように見えるものは、通常、次の段落の **通常のテキスト** であり、Marked はそれを本文テキストとして保持します。

埋め込み画像と RTFD バンドル内の画像 (たとえば、Pages エクスポートの `pastedGraphic.png`) は、`~/Library/Caches/Marked/Watchers/` の下のキャッシュ フォルダーにコピーされます。エクスポートするまで、プレビュー画像のパスはそれらのキャッシュされたファイルを指します。

Marked は、画像ファイル名を代替テキストまたは MultiMarkdown 図のキャプションとして**使用しません**。文書にテキストを入力しない限り、画像の下に「`pastedGraphic.png`」は表示されません。

## エクスポートとその他の機能

変換後、Marked はドキュメントを他のコンパイル済みソースと同様に扱います ([Scrivener](Scrivener_Support.html) や [DOCX](Working_with_DOCX.html) と同様)。エクスポート、統計、およびほとんどのプレビュー機能は、Watchers キャッシュに保存されている生成された Markdown に対して実行されます。

## 制限事項

変換品質はソース アプリケーションによって異なります。一般的に:

- **見出し**は、Word/Pages のアウトライン スタイルではなく、フォント サイズから推測されます。
- **複雑なレイアウト** (テキスト ボックスの位置決めのみに使用される複数列のテーブル) は、Markdown にきれいにマップされない場合があります
- RTF の **方程式** はプレビューではサポートされていません (マークダウン計算については [MathJax](MathJax.html) を参照してください)
- **レガシー `.doc`** (バイナリ Word) はサポートされていません。最初に DOCX または RTF として保存します

ファイルを保存せずに一度だけ貼り付ける場合は、[クリップボードプレビュー](Opening_Files.html#from-the-clipboard)を使用してください。
