# <%= @title %>

Marked の Mac App Store (MAS) バージョンは、セキュリティ上の理由から特定の操作を制限するサンドボックス環境で実行されます。これは、特に外部バイナリまたはスクリプトでカスタム プロセッサを使用する場合、一部の機能に影響を与える可能性があります。

## 一般的なサンドボックスの制限

### コマンドバイナリを実行する

ユーザーが遭遇する最も一般的な問題は、外部バイナリを MAS バージョンで直接実行できないことです。これは以下に影響します。

- **外部プロセッサ** (Pandoc、Kramdown、その他のコマンドライン ツールなど)
- 外部バイナリに依存する **カスタム スクリプト**
- サンドボックスからアクセスできない **システム ユーティリティ**

### 回避策

#### バイナリを App Bundle にコピーする

MAS バージョンで Pandoc などの外部プロセッサを使用する必要がある場合は、バイナリをアプリ バンドルにコピーできます。

1. Marked.app を右クリック → **パッケージの内容を表示**
2. **コンテンツ/リソース/** に移動します
3. `bin` フォルダーが存在しない場合は作成します
4. バイナリ (例: `pandoc`) をこの `bin` フォルダーにコピーします
5. 実行可能にします: `chmod +x` バイナリ
6. 「コマンドの実行」アクションでは、次のように参照します。
   - `YOUR_BINARY_NAME [arguments]`
   - またはフルパスを使用します: `/Applications/Marked.app/Contents/Resources/bin/YOUR_BINARY_NAME [arguments]`

**注意**: 外部シバンを含むスクリプト (`/opt/homebrew/bin/python` を指す Python スクリプトなど) は、バンドルにコピーされるとシステム インタプリタを通じて自動的に実行されます。 MAS バージョンでは、バイナリ ファイルではなく、実際には Python または Ruby スクリプトであるバイナリの実行に依然として問題が発生する可能性があります。

**重要**: アップデートによりバンドル全体が置き換えられるため、アプリをアップデートするたびにバイナリを再コピーする必要があります。

#### 埋め込みスクリプトの使用

外部コマンドを実行する代わりに、カスタム ルールで **埋め込みスクリプトの実行** アクションを使用できます。これにより、Marked のコード エディターでスクリプトを直接作成できるようになり、サンドボックス内で使用できるシステム インタープリターにアクセスできます。

## 非サンドボックス バージョンへの切り替え [クロスグレード]

外部バイナリを頻繁に使用する必要がある場合、またはその他のサンドボックス制限に遭遇する場合は、Marked の非サンドボックス バージョンに切り替えることをお勧めします。非サンドボックス バージョンにはそのような制限はなく、インストールしたコマンド ライン ツールやスクリプトを実行できます。

### サブスクリプション ユーザー向け

アクティブな Mac App Store サブスクリプションをお持ちの場合:

1. システム設定 → Apple ID → メディアと購入 → サブスクリプションで **MAS サブスクリプションをキャンセル**
2. [https://markedapp.com](https://markedapp.com) から **無料トライアルをダウンロード**
3. アプリまたは Web サイトから直接 **Paddle サブスクリプションを開始**

Paddle バージョンは、サンドボックスの制限なしで同じ機能を提供します。

### 永久ロック解除保持者の場合

Mac App Store を通じて永久ロック解除ライセンスまたは永久ライセンスを購入した場合は、[電子メールで開発者](mailto:marked@brettterpstra.com?subject=Marked%20License%20Crossgrade&body=Please%20include%20your%20UUID%20%28Help-%3ECopy%20UUID%20in%20Marked%29%20in%20this%20email%20for%20receipt%20verification.)無料の生涯 Paddle ライセンスをリクエストするには、

**重要**: 購入を確認するには、メールに次のいずれかを含めてください。

- **ユーザー識別子** (UUID) - **[ヘルプ]->[UUID をコピー]** を使用してクリップボードにコピーし、メールに貼り付けます
- App Store のレシートに記載されている **トランザクション ID** (これは App Store アプリの購入履歴で確認できます)

Mac App Store は開発者に電子メール アドレスを提供しないため、サーバーに保存されているトランザクション ID またはユーザー ID を使用して購入を確認します。この情報を含めると、購入を迅速に確認し、無料の Paddle ライセンスを生成することができます。

### アプリのバージョンを設定する

あるいは、Setapp サブスクリプションをお持ちの場合は、Marked の Setapp バージョンを使用できます。これもサンドボックス化されておらず、システム リソースに完全にアクセスできます。
