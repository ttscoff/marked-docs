# <%= @title %>

2 つのお気に入りの筆記具を組み合わせて使用します。

> Marked は引き続き Scrivener 2.0 ファイルを読み取ることができますが、開発は Marked 2.5.11 以降のバージョン 3 に集中します。

## Scrivener 3.0 の基本

Scrivener プロジェクト (.scriv) を [Marked] にドラッグすると、コンパイルされてプレビューされます。 Scrivener で .scriver ファイルを開くオプションを選択した場合 (上記)、ファイルを Marked にドラッグすると、Marked は Scrivener も起動します。

他のドキュメントと同様に、Scrivener ファイルへの変更は保存時にライブで更新されます。また、Scrivener ドキュメントが Marked のフォアグラウンドにある場合、{% kbd cmd E %} はそれを Scrivener で開きます。

## バインダー文書のフィルタリング

Marked で Scrivener プロジェクトを開くと、選択したバインダー ドキュメントからのみプレビュー コンテンツが構築されます。フィルタリングは `.scriv` ファイルに対して常に有効です。フィルター パネルは、含まれるものを変更するための便利な方法にすぎません。

**[校正] > [Scrivener ドキュメントのフィルター]** からパネルを開きます ({% kbd opt-cmd-f %})。パネルが表示されている間、メニュー項目にはチェックマークが表示されます。パネルを閉じても、フィルタリングがオフになったり、選択がリセットされたりすることはありません。

フィルター パネルには、プロジェクトのバインダー セクション (ドラフト、リサーチ、およびゴミ箱を除くその他のトップレベル バインダー) がリストされます。各セクションはデフォルトで折りたたまれています。セクションを展開すると、そのドキュメントとフォルダーがアウトラインで表示されます。

- **テキスト ドキュメント**をオンまたはオフにして、プレビューに含めるかプレビューから除外します。
- **フォルダー**行をクリックして、その子孫をすべて選択または選択解除します。チェックボックス内のダッシュは、一部の子孫のみが選択されていることを意味します。
- Scrivener で **コンパイルに含める** がオフになっている行は灰色で表示されますが、マーク付きでチェックを入れてプレビューすることができます。
- **画像、PDF、およびその他のテキスト以外**のバインダー項目がリストに表示されますが、選択できません。
- **欠落している RTF** ファイルが依然として表示されます。 Marked が開いているときに Scrivener にコンテンツを追加すると、他の Scrivener の変更と同様に、保存時にプレビューが更新されます。

プロジェクト全体に対して、パネルの上部にある [**すべて選択**] および [**すべて選択を解除**] を使用します。各セクションのヘッダーには、そのセクションのみの [**すべて**] ボタンと [**なし**] ボタンがあります。 **すべて選択を解除** は、ドキュメントがチェックされていないことを意味します。

何も選択されていない場合、プレビューにはフィルター パネルを開くためのリンクを含む短いメッセージが表示されます (`x-marked://scrivenerfilter`)。その URL をスクリプトまたは他のアプリで使用して、Marked の前面の Scrivener ドキュメントのパネルを表示できます。

チェックボックスの選択は Scrivener プロジェクトごとに (`.scrivx` ファイル内のプロジェクト ID によって) 保存され、次回そのプロジェクトを Marked で開くときに復元されます。初めてプロジェクトを開いたとき、Marked は、**コンパイルに含める** フラグが Yes (または未設定で、Scrivener が Yes として扱う) である **ドラフト** バインダー ドキュメントのみを選択します。研究やその他のバインダーはチェックされずに開始されます。コンパイル除外されたドラフト項目はチェックなしで開始されますが、リスト内では選択可能なままです。

Scrivener 2 プロジェクトでは、フィルター パネルに **ドラフト** バインダーのみが表示されます。 Scrivener 3 プロジェクトには、ゴミ箱を除くすべてのバインダーが含まれています。

フィルター パネルは、**単語の繰り返しの視覚化** などの他のツールと一緒に開いたままにすることができます。チェックボックスを変更すると、少し遅れてプレビューが再コンパイルされます。大規模なプロジェクトがまだコンパイル中の場合、Marked は進行中のインポートをキャンセルし、新しい選択内容から再度開始します。

## Scrivener タイトルの Markdown ヘッダー

Marked は、Scrivener ファイルのページに基づいて階層的な Markdown ヘッダーを作成することもできます。これを有効にするには、上に表示されているオプションをチェックするだけです。

## MultiMarkdown メタデータ

Draft フォルダー内の最初のドキュメントの名前が「metadata」である場合、プレビュー ドキュメントの先頭で MultiMarkdown メタデータとして扱われます。 「Scrivener Titles からの Markdown ヘッダー」設定 (上記) に関係なく、このセクションにはヘッダーは挿入されないため、MultiMarkdown プロセッサはそれをメタデータとして読み取り、それに応じて置換およびエクスポート オプションを許可できます。

プロセッサが YAML を処理できるプロセッサである場合は、このファイルを YAML 形式にすることができます。

`metadata` ドキュメントを使用しない場合、Marked は、Scrivener が MultiMarkdown にエクスポートするのと同じ **Title** および **Author** フィールドを使用して、プロジェクトのコンパイル設定 (`Settings/compile.xml`) から MultiMarkdown メタデータを先頭に追加することもできます。これはデフォルトで有効になっています (設定キー `scrivenerCompileMetadata`)。カスタム メタデータ フィールドは、プロジェクトの **MMD メタデータ** コンパイル設定に表示される場合にのみ含まれ、ドキュメントごとのカスタム フィールドからは含まれません。

## リンク

外部 (HTTP) リンクの場合は、Markdown 構文または Scrivener のリンク形式を使用できます。 Marked は、処理前に Scrivener 形式を Markdown に変換します。

## コメント

Marked は、文書内でインラインで作成されたコメントと脚注を処理できます。

## テーブル

Marked は基本的な Scrivener テーブルを変換できます。ただし、出力にテーブルを含めたい場合は、[MultiMarkdown テーブル形式](https://github.com/fletcher/MultiMarkdown/wiki/MultiMarkdown-Syntax-Guide#tables) で行うのが最善です。 ([TableFlip](http://tableflipapp.com/) というアプリを使用すると、これらを簡単に生成できます。)

## Scrivener の追加機能

基本的なコンパイルおよびプレビュー機能に加えて、Marked は Scrivener 固有の規則もいくつかサポートしています。まず、Scrivener ドキュメントで、インラインまたはスタンドアロン テキストのブロックで「書式を保持」を使用すると、プレビューでコード ブロックに変換されます。

Marked は Scrivener からの _inline_ 脚注も読みます。段落内または段落の末尾に脚注を入力すると、プレビューで MultiMarkdown 脚注に変換されます。

## Scrivener ドキュメントでの画像の使用

画像は Scrivener ドキュメントに埋め込んだり、Markdown 画像構文で参照したりできます。イメージタグの Markdown バージョンは `![alt text](path/to/image.ext "optional title/description")` です。参照フォーマットも同様に使用できます。

    ![代替テキスト][img1]

    [img1]: /path/to/image.ext "オプションの説明"

プレビュー内の HTML 出力のベース パスは、Scrivener ドキュメントを含むフォルダーに設定されます。したがって、作業ドキュメントと同じフォルダーに画像を配置すると、画像に直接アクセスできるようになります。 Scrivener ドキュメントが `~/Desktop/TestDoc.scriv` にあり、「testimage.png」という名前の画像が `~/Desktop/testimage.png` にある場合、次を使用して画像をドキュメントに追加できます。

    ![テスト画像](testimage.png)

ドキュメントの親フォルダーに基づく相対パスは機能しますが、絶対パスを使用するとどこからでも画像にアクセスできますが、HTML 出力には移植できない可能性があります。

## セキュリティに関する注意事項

Marked で .scriv ファイルを開くと、~/Library/Application Support/Marked にキャッシュ フォルダーが作成されます。これは保護されたフォルダーではないため、元のドキュメントが暗号化されたディスク上にある場合、またはその他の方法で保護されている場合、そのコンテンツはキャッシュ内で暗号化されないことに注意してください。保護を限定するには、Spotlight のプライバシー設定に ~/Library/Application Support/Marked を追加することで、このキャッシュが Spotlight に表示されないようにすることができます。

クリップボードのプレビュー、Wiki リンク、スクリプト、およびアプリごとのガイドの完全なリストについては、[追加のアプリ統合](Additional_Application_Support.html) を参照してください。
