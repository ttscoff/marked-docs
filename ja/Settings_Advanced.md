YAML と Pandoc メタデータ
: - __Ignore:__ ドキュメント内に存在するものをそのまま残します。プロセッサまたはプリプロセッサが実際に YAML を使用する場合に便利です。
: - __レンダリング前に削除__: YAML ブロックが削除されます
: - __MMD メタデータとして扱う:__ YAML または Pandoc メタデータ ブロックを MultiMarkdown 形式に変換します。

MultiMarkdown メタデータ ヘッダーを削除する
: これを有効にすると、ドキュメントの上部にある MultiMarkdown メタデータがレンダリング前に削除されます。
: これは、レンダリングされたドキュメントにメタデータが表示される非 MultiMarkdown プロセッサを使用している場合に便利です。メタデータは削除前に読み取られるため、マーク固有の構文は引き続き認識されます。

ホストされている画像をキャッシュする
: Marked は、画像の変更を監視し、即座に更新するため、デフォルトではプレビュー内の画像をキャッシュしません。 Web URL 経由で参照される画像を使用している場合は、それらの画像のキャッシュをオンにして、低速接続でのレンダリングを高速化できます。

可読性の統計を分析する
: [統計](Document_Statistics.html) を計算したくない場合は、その処理が無効になります。これにより、非常に大きなドキュメントのパフォーマンスが向上する可能性があります。文字数、単語数、文数のカウントは引き続き機能します。

システム全体の検索ペーストボードを使用する
: このオプションを使用すると、Marked は別のエディターで最近使用した検索語をすべて取得できるようになり、Marked で検索されたものはすべて他のアプリケーションでも検索になります。無効にするとMarkedの検索機能が独立して動作します。

選択範囲を使用して検索するには {% kbd cmd E %} を使用します
: デフォルトでは、{% kbd cmd E %} はデフォルトのエディタを開きます。このオプションが有効な場合、{% kbd cmd E %} はほとんどのテキスト編集アプリケーションと同様に機能し、選択したテキストを検索に使用し、{% kbd opt cmd E %} でエディターで開くをトリガーできます。

デバッグモード
: デバッグログを有効にします。これは、独自のトラブルシューティングや問題を報告するときに使用してください。 [__ヘルプ]->[デバッグ ログを開く]__ を選択してアクティビティを表示します。
: _All_ に設定すると、情報メッセージ、警告、エラー メッセージが表示されます。エラーのみ、またはエラーと警告を表示するように設定することもできます。

設定のバックアップ
: 設定を JSON ファイルにバックアップして、設定を復元したり、新しいマシンに移植したりするために使用できます。
