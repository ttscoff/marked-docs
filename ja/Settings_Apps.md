### 一般設定

テキストエディタ
: {% kbd cmd E %} と入力すると、現在のドキュメントを受け取るテキスト エディターを選択します。

新しいファイルを自動的に編集する
: 「新規ファイル」コマンドを使用する場合、このオプションは、選択した外部エディタで作成されたファイルを自動的に開きます。

テキスト ファイルへのリンクは次の場所で開く必要があります。
: プレビュー ウィンドウでリンクをクリックしてローカル テキスト ファイルにアクセスしたときの Marked の動作を決定します。

画像エディター
：ローカル画像を右クリックして「画像編集」を選択したときに開くアプリケーションを選択します。

画像注釈/マークアップエディター
: ローカル画像を右クリックして「画像に注釈を付ける」を選択したときに開くアプリケーションを選択します。

エディターが選択されていない場合、編集または注釈を付けるときに、「マーク」にはインストールされているアプリケーションのメニューが表示されます。メニューには、Mac にある一般的なマークダウン、画像、注釈ツール、`/Applications` から任意のアプリを選択する **その他…** オプション、選択したものをデフォルトとして保存する **常にこのアプリを使用する** (デフォルトで有効) が含まれています。 {% prefspane Apps %} の各選択コントロールの横にあるクリア ボタン (X の付いた丸) を使用して、選択を削除し、ピッカーに戻ります。

### アプリケーション固有の設定

デフォルトでコメントと注釈を表示する
: チェックすると、Scrivener、Fountain、Word、および CriticMarkup ドキュメントで作成された注釈がプレビューで強調表示されます。完全に非表示にするにはチェックを外します。これらは、文書の読み取り中に{% appmenu Gear, Proofing ({{ctrl}}{{cmd}}C)%} メニューから切り替えることもできます。
: コメントが有効になっている場合、コメントと脚注がサイドバーに表示されます。コメントの上にマウスを置くと、ドキュメント内のコメントが存在する場所が示されます。

#### ドキュメント

[(DocC サポートに関する情報)](DocC_Support.html)

DocC イメージ参照を解決する
: `.docc` ドキュメント カタログ内で、`~dark` および `@2x` バリアントを含む、カタログ `Resources` フォルダー内の画像への拡張子なしの `![](ImageName)` 参照を解決します。

暗い画像および @2x の画像バリアントを解決する
: ファイル拡張子 (`images/icon.png`) を持つローカル画像の場合、同じフォルダー内のコンパニオン `~dark` および `@2x` ファイルを検出し、応答性の高い `<picture>` マークアップを生成します。 DocC カタログだけでなく、あらゆる Markdown または HTML ドキュメントでも機能します。 [画像バリエーション](Image_Variants.html)をご覧ください。

#### フックマーク

画像とリンク内のhook:// URLを解決する
: Marked は、Hookmark によって作成された URL を読み取り、ディスク上のパスに解決できます。

#### Leanpub/GitBook

Leanpub サポートを有効にする
: `Book.txt` という名前のファイルをインデックス ファイルとして解釈し、特殊な Leanpub 構文を処理します。

GitBook サポートを有効にする
: `SUMMARY.md` という名前のファイルを GitBook ドキュメントのインデックス ファイルとして解釈します。

#### Markdownifier

インラインリンクを使用する
: Markdownifier によって作成された Markdown ドキュメントは、参照リンクの代わりにインラインを使用します。

#### 火星編集

投稿タイトルをマークダウンヘッダー (h1) として含めます
: 選択した投稿のタイトルをマークダウン ヘッダーとして使用します。

メタデータをテーブルとして表示する
: 有効にすると、カテゴリやタイトルなどのメタデータがマークダウン テーブルとしてプレビューに表示されます。

#### フォルダー

これらの拡張機能のみをプレビューする
: フォルダーを開くと、Marked は最後に変更されたドキュメントを検索します。デフォルトでは、`md`、`mmd`、`html` などの拡張子が使用されます。ここのリストはデフォルトをオーバーライドします。

#### 書士

[(行政書士サポートのご案内)](Scrivener_Support.html)

ドキュメントのタイトルを Markdown ヘッダーとして含める
: ページとネストされたページのタイトルを解析して、階層的な Markdown ヘッダーを作成します。

コンパイルメタデータ (タイトル、作成者) が欠落している場合は追加します。
: Scrivener プロジェクトに `metadata` ドキュメントまたは既存の MultiMarkdown ヘッダーがない場合、プロジェクトのコンパイル設定 (`Settings/compile.xml`) からタイトルと作成者を先頭に追加します。

Marked で開かれた .scriv ファイルを Scrivener で開く
: Scrivener ドキュメントを Marked で開くと、自動的に Scrivener でも開きます。

#### 単語

変更追跡を変換 <-> CriticMarkup
: 有効にすると、Word ドキュメントの変更追跡はインポート時に CriticMarkup に変換され、DOCX ファイルのエクスポート時に CriticMarkup が Word の変更追跡に変換されます。

#### マインド マップ/アウトライン {#MindMapsOutlines}

マーメイド マインド マップとして埋め込む
: 各チェックボックスは、含まれる 1 つの形式を制御します。 **オン**の場合、含まれているファイルはマーメイド マインド マップ図 (整頓されたツリー レイアウト) に変換されます。 **オフ**の場合、Marked はその形式の代替を使用します。
: **マインド マップ** — iThoughts X (.itmz)、MindManager (.mmap)、FreeMind (.mm)。オンの場合: マーメイド マインド マップ。オフの場合: iThoughts はプレビュー画像を埋め込みます。 MindManager と FreeMind は、ネストされた Markdown リストに変換されます。
: **OPML ファイル** (.opml)。オンの場合: マーメイド マインド マップ。オフの場合: ネストされたマークダウン リスト。
: **OmniOutliner のアウトライン** (.ooutline)。オンの場合: マーメイド マインド マップ。オフの場合: ネストされたマークダウン リスト (または該当する場合はチェックリスト)。
: **自転車の概要**。オンの場合: マーメイド マインド マップ。オフの場合: ネストされたマークダウン リスト。
