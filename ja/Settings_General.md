### ウィンドウ

新しいウィンドウを一番上に表示する
: 新しいウィンドウが他のアプリケーションの上に「フロート」するように自動的に設定されます。

更新時にウィンドウを上げる
: 監視しているファイルで変更が検出されると、マーク付きをアクティブにせずに、そのドキュメントのプレビュー ウィンドウがデスクトップ上の他のウィンドウの上に表示されます。

背景が半透明
: ウィンドウにフォーカスが当たっていないときは、ウィンドウをフェードさせます。スライダーを使用して不透明度を設定します。

大きなドキュメントではメモリを大量に消費する機能を無効にする
: ドキュメントが 100k を超える場合は、折りたたみ可能な見出しなど、プロセッサを大量に使用する一部の機能を無効にします。

新しいドキュメントが開きます
: **windows**、**tabs**、または **automatic** を選択します (タブ移動については macOS システム設定に従います)。タブを使用する場合は、{% kbd cmd shift [/] %} と [クイックオープンパネル](Quick_Open.html) を使用して移動します。

更新されたドキュメントを前面に表示
: プレビューが更新されると、そのタブを選択するか、**マークされた範囲内のみ**でそのウィンドウを最前面に並べます。別のアプリケーション (例: テキスト エディター) が最前面にある場合、Marked はバックグラウンドに残ります。正しいタブが選択されているため、Marked に戻ったときにすぐに使用できるようになります。マークをアクティブにせずにすべてのアプリケーションの上にプレビューを表示するには、代わりに **更新時にウィンドウを表示** を使用します。

フォーカスを前のアプリに戻す
: 有効にすると、更新/更新の実行アクションにより Marked がフォアグラウンド フォーカスを取得する場合、キーボード フォーカスは更新前に最前面にあったアプリ (テキスト エディターなど) に戻ります。無効にすると、Marked はこのフォーカスのハンドオフを実行しません。 Marked が最前面にならない場合、このオプションは効果がありません。

### ステータスバー

スタイルピッカーを表示
: プレビュー ウィンドウの下部バーにスタイル ピッカーを表示します。

単語数を表示
: プレビュー ウィンドウの下部バーに単語数 (および統計ボタン) を表示します。

単語数に含まれないもの
: 単語数の計算では、次の組み合わせを無視できます。

- **脚注/引用**
- **ブロック引用符**
- **インデントされたコード ブロック** (フェンスで囲まれたコード ブロックは常に除外されます)
- **画像キャプション**

### ショートカット

ショートカット フィールドをクリックして、イベントをトリガーするホットキーの組み合わせを記録します。

マーク付きのアクティブ化
: いずれかのアプリケーションでこのホットキーを押すと、マーク付きに切り替わります。

最初のウィンドウを上げる
: 現在のアプリケーションを離れることなく、最前面 (最後にアクティブな) マークされたプレビュー ウィンドウを最前面に上げます。

アクションパレットを開く
: マークがアクティブなときに[クイックアクション](Quick_Actions.html)コマンドパレットを開きます。このショートカットは {% appmenu File, Quick Actions… %} に適用され、Marked 内でのみ機能します (他のアプリケーションからは機能しません)。

アラートのリセット
: 以前に閉じた警告ダイアログを復元して、再度表示できるようにします。
