### プレビュー動作

ミニマップナビゲーションを有効にする
:「0」を押したときに表示されるドキュメントのビジュアルマップを生成します。大きなドキュメントをレンダリングするときに短時間の遅延が発生する可能性があります。

見出しはセクションを折りたたむ
: 見出し要素をクリックすると、その要素と次の見出しの間のセクションが折りたたまれます。

{% kbd cmd %}‑クリックが必要です
: これをチェックすると、Command キーを押しながらクリックした場合にのみ見出しが折りたたまれたり展開されたりします。

同期プレビューとソーススクロール
: エディターがサポートしている場合は、ソース文書内の対応する位置に一致するようにプレビューをスクロールしたままにしてください。

スクロール位置と速度読み取りを同期する
: [速読](Speed_Reading.html)オーバーレイをプレビューのスクロール位置に合わせてください。これは、Speed Read オーバーレイから切り替えることもできます。

### スクロールして編集します

スクロールして編集します
: プレビューを更新するときに、Marked はドキュメントが変更された最初のポイントを特定し、そこまで自動的にスクロールします。これにより、編集中のドキュメント内の現在の位置とプレビューが常に同期されます。最新の編集マーカーは、最後の更新以降のドキュメント内の最初の違いです。 「差分順序を逆にする」を有効にすると、ドキュメント内の最後の差分 (上から下) が最新の編集であるとみなされます。

最新の編集マーカーを表示
: 最初に検出された編集の位置に小さな赤いマーカーが表示されます。これをオフにすると非表示になります。 (編集するには **スクロールする必要があります**。)

すべての差分マーカーを表示
: これを有効にすると、前回の更新と現在のコンテンツの間のすべての違いがガターで強調表示されます。 <kbd>e</kbd> (進む) と {% kbd shift E %} (戻る) を使用して、編集内容をビューの中央までスクロールして移動できます。

差分順序を逆にする
: これが有効な場合、差分は逆の順序 (下から上) でマークされます。これはナビゲーションに影響するため、<kbd>e</kbd> は上に移動し、{% kbd shift E %} は下に移動します。 「最新の編集」がドキュメント内の最後の差分になります。

### 追加機能

目次はスクロール位置を追跡します
: 目次は現在のセクションを強調表示します。

テキスト選択のポップアップ統計
: 選択したテキストの単語数ポップアップを表示します。

リンクポップオーバーを有効にする
: 外部リンクをクリックして押したまま放すと、ポップアップ メニューが表示されます。

更新時に URL を自動的に検証する
: ドキュメントのロードおよび更新時に URL を検証します。エラーがある場合のみ結果を表示します。
: これは、ドキュメントが更新されるたびに [リンク検証](Link_Validation.html) を実行します (リンクの数が多い場合、これはプロセスが遅くなる可能性があるため、避けるべきです)。

### Wiki へのリンク

[[Wiki リンク]] を変換する
: `[[wiki link]]` 構文に対して Marked の [wiki ナビゲーション](Wiki_Navigation.html) を有効にします。

デフォルトの内線番号
: 拡張子を含まない Wiki リンクを解決するときに Marked が使用するファイル名拡張子 (例: `md`)。

### 外観

ダークモード
: すべてのウィンドウを「ハイ コントラスト」モードでダーク クロムで表示し、利用可能な場合は現在のスタイルの反転バージョンを表示します (カスタム スタイルには適用されない場合があります)。

システム設定の一致
: macOS ダーク モードがシステム全体でオンまたはオフになったときに、ダーク モードを自動的に切り替えます。

ウィンドウタイトルにフルパスを表示
: 有効にすると、Marked はウィンドウ タイトルに現在のドキュメントへのフル パスを表示します。
