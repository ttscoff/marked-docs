### マークダウンを処理する

デフォルトのマークダウンプロセッサ。 CommonMark プロセッサは GitHub ユーザーに好まれ、MultiMarkdown はライターに最適で、Discount と Kramdown は特殊な目的を持っています。マークは、構文間のいくつかの違いを補います。詳細については、「__Help->Markdown Reference__」を参照してください。

カスタムルール
: [カスタム ルール] ボタンをクリックしてルール エディタを開き、一致基準に基づいて実行するさまざまなプロセッサとドキュメント変換を指定できます。詳細については、「カスタムプロセッサー」(Custom_Processor.html)を参照してください。

新しいドキュメントはカスタムを使用します
: チェックすると、新しいドキュメントは、ドキュメントごとの設定を必要とせずに、カスタム ルールから保存した**プリプロセッサ**や**プロセッサ**を自動的に使用します。

フルディスクアクセス
: [**許可**] をクリックして、カスタム プロセッサまたはより広範なファイル アクセスが必要なその他の機能を使用する場合に、サンドボックスの外でファイルを読み取るためのアクセス許可をマークに付与します。

プロセッサ間の違いを調べるには、[Markdown Dingus](Markdown_Dingus.html) を確認してください。

### HTML

見出しに ID を生成する
: h1～h6タグの内容に基づいてヘッダIDを生成します。

ランダムな脚注 ID を使用する
: 複数の文書が 1 ページに表示される場合の競合を避けるために、ランダムな脚注 ID を生成します。

HTML内のMarkdown処理
: デフォルトでは、HTML タグ内の Markdown は通常無視されます。このオプションは、Marked にブロック要素内での処理を強制的に継続させます。マークアップによっては問題が発生する可能性があることに注意してください。

### レンダリング

段落内の改行を保持する
: 段落テキスト内の改行を尊重し、前の行と連結するのではなくハード改行に置き換えます。

GitHub チェックボックスをレンダリングする
: リストにチェックボックスを作成するために、`- [ ]` と `- [x]` をレンダリングします。チェックボックスはプレビュー用にレンダリングされますが、Marked 内では機能しません。

GitHub をレンダリングします:絵文字:
: プレビューで `:name:` ショートコードを GitHub スタイルの絵文字としてレンダリングします。

\#テキストはタグです
: ハッシュタグ (`#` のすぐ後にスペースなしのテキストが続く) を見出しではなくタグとして解釈できるようにします。この機能は Bear ユーザーのデフォルトです。

スタイルタグ
: **#Text is tag** が有効な場合、認識されたタグをカプセル スタイルで表示します。タグは{% appmenu {{gear}}, Proofing, Show Comments %}から表示または非表示にすることができます。

![#Text タグが有効です][textistag]

`==highlight==` と `~~delete~~` をレンダリングします
: このオプションが有効な場合、`==highlight==` テキストは HTML `<mark>` タグとしてレンダリングされ、スタイルによって変更されない限り、黄色のハイライトとして表示されます。さらに、`~~delete~~` 構文は `<del>` タグを使用してレンダリングされます。

: __Render as CriticMarkup__ が有効な場合、`==highlight==` および `~~delete~~` 構文は CriticMarkup に変換され、元のビュー、マークアップ ビュー、編集済みビューで表示できます。

`~text~` をアンダースコアとしてレンダリングします
: このオプションが有効な場合、単一のチルダで囲まれた `~text~` が下線付きでレンダリングされます。これは添え字の MultiMarkdown 構文と競合するため、デフォルトでは無効になっています。
