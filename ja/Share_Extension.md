# <%= @title %>

Marked には、システムの共有メニューに表示される macOS **共有拡張機能**が含まれています。アプリを切り替えたり URL を手動でコピーしたりせずに、ファイルや選択したテキストを Marked に送れます。

共有拡張機能は **Marked 3 に同梱**されています。別途ダウンロードやインストールは不要です。Direct、Mac App Store、Marked Pro、Setapp 各ビルドに含まれます。

## 動作

共有メニューで **Marked** を選ぶと、Marked がすぐに開きます。中間の作成ウィンドウはありません。

### ファイルを共有

**Finder**（またはファイルを共有できるアプリ）で **共有 → Marked** を選びます。

Marked はファイルパスを受け取り、他と同じ `x-marked-3://open` URL ハンドラで開きます。Dock アイコンへドラッグしたり {% appmenu File, Open... ({{cmd}}O) %} で選んだときと同様に開きます。

ファイル URL、ローカルファイル、送信元アプリが提供する Web URL に対応しています。

### 選択テキストを共有

**TextEdit**、**Safari**、**Mail** などでテキストを選択し、**共有 → Marked** を選びます。

Marked はテキストをクリップボードに置き、`x-marked-3://paste` ハンドラで **一時プレビュー**を開きます。{% appmenu File, New, Clipboard Preview ({{shift}}{{cmd}}V) %} と同じ未保存プレビューです。後から {% appmenu File, Save Transient Preview %} で保存できます。

プレーンテキスト、HTML、RTF、Markdown は送信元アプリが提供する場合に対応しています。

基になるコマンドの詳細は [URL Handler](URL_Handler.html) を参照してください。

## 共有メニューの使い方

### Finder から

1. Markdown またはテキストファイルを右クリック（または選択して Finder ツールバーの **共有** をクリック）。
2. 共有メニューで **Marked** を選びます。

**Marked** が表示されない場合は、下の [共有拡張機能を有効にする](#enable-the-share-extension) を参照してください。

### テキスト選択から

1. プレビューしたいテキストを選択します。
2. アプリの **共有** メニューを開きます（メニューバー、ツールバー、またはコンテキストメニュー）。
3. **Marked** を選びます。

Marked が起動（または前面に表示）し、共有内容のプレビューが開きます。

## 共有拡張機能を有効にする

Marked を `/Applications`（または通常のアプリケーションフォルダ）にインストールし、少なくとも一度起動してからでないと、macOS は共有拡張機能を一覧に表示しません。

### システム設定で Marked をオンにする

1. **システム設定**を開きます。
2. **一般 → ログイン項目と拡張機能**へ（macOS によっては **プライバシーとセキュリティ → 拡張機能**）。
3. **拡張機能**（または拡張機能横の **ⓘ**）をクリックします。
4. **共有**（または **Sharing**）を選択します。
5. **Marked** を有効にします。

### アプリの共有メニューに Marked を追加

拡張機能がシステム全体で有効でも、各アプリは表示する共有先を選べます。

1. 共有に対応したアプリを開きます（Finder と TextEdit が手軽です）。
2. **共有**メニューを開きます。
3. **拡張機能を編集…**を選びます（古い macOS では **その他…** や **拡張機能環境設定…** の場合あり）。
4. **共有**の下で **Marked** にチェックを入れます。
5. 必要なら **Marked** をリストの上へドラッグしてアクセスしやすくします。

多くのアプリでは変更はすぐに反映されます。

## 共有に Marked が表示されない場合

W> 共有拡張機能は Marked 3.1.9 以降で利用できます。少なくともそのバージョンに更新してください。

次の手順を順に試してください。

1. インストールまたは更新後、**Marked を一度起動**します。更新直後は終了して再度開いてください。
2. 上記のとおり **システム設定で拡張機能が有効**か確認します。
3. 共有元アプリで **共有メニューをカスタマイズ**します。macOS は使わない共有先を非表示にすることがあります。
4. **Marked が複数ある場合：** Direct 版と Mac App Store 版の両方があると、実行中のコピーだけが拡張機能を登録します。使わない方を削除または名前変更し、使う方を起動してください。
5. 更新後も表示されない場合は **Mac を再起動**します。macOS は共有拡張機能の登録をキャッシュします。
6. Xcode やディスクイメージから手動コピーしたビルドを試している場合は **`/Applications` に Marked を再インストール**してください。共有拡張機能は `Marked.app/Contents/PlugIns/` に埋め込まれている必要があります。

## ヒント

- 共有拡張機能は、Web の抜粋、メールの段落、メモなどをファイルを作らずに素早くプレビューするのに向いています。
- ページ全体やブラウザの複雑な選択には、[ブラウザ拡張機能](Using_the_Browser_Extensions.html)の方が操作しやすい場合があります（セクション選択、Markdownify URL など）。
- 共有したファイルは通常の Marked ドキュメントとしてファイル監視付きで開きます。共有テキストは保存するまで一時プレビューとして開きます。
