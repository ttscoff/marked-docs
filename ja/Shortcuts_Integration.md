# <%= @title %>

マークには、ファイルを開く、プレビュー スタイルの変更、ドキュメントのエクスポートのためのネイティブ **ショートカット** アクション (アプリ インテント) が含まれています。これらのアクションは、**マーク付き**を検索するとショートカット アプリに表示され、**macOS 13 (Ventura)** 以降で利用できます。

完全なオブジェクト モデルを使用したスクリプトベースの自動化については、[AppleScript サポート](AppleScript_Support.html) を参照してください。シェルからの URL ベースのワークフローについては、「URL ハンドラー」(URL_Handler.html) を参照してください。

## アクションの検索

1. **ショートカット** アプリを開きます。
2. 新しいショートカットを作成するか、既存のショートカットを編集します。
3. アクション ライブラリで **マーク付き** を検索します。

アクションは **ドキュメント** と **エクスポート** にグループ化されています。 Marked は、「Marked でファイルをエクスポート」や「Marked で開く」などの Siri フレーズを登録して、簡単にショートカットできるようにします。

## アクションの概要

|アクション |目的 |
| --- | --- |
| **マークされたファイルを開く** |プレビュー ウィンドウで Markdown ファイルまたはテキスト ファイルを開きます。 |
| **プレビュー スタイルを設定** |開いているドキュメントのプレビュー テーマを変更します (または最初にファイルを開きます)。 |
| **ファイルを開いてエクスポート** |ファイルを開いて別の形式にエクスポートします。 |
| **ドキュメントのエクスポート** |開いているドキュメント (フロント ウィンドウまたは特定のファイル) をエクスポートします。 |
| **開いているドキュメントをエクスポート** |現在マークで開いているすべてのドキュメントをエクスポートします。 |

すべてのエクスポート アクションは、エクスポートされたファイル (複数可) をショートカット **ファイル** 出力として返すため、それらを次のステップ (メール、ファインダー、別のアプリ) に渡すことができます。

## マークされたファイルを開く

**パラメータ:** **ファイル** -- (Finder、共有シート、または前のショートカット手順から) 開くドキュメント。

マークすると、プレビュー ウィンドウでファイルが開きます。他の作業を行う前に、Marked でプレビューまたは編集したい場合にこれを使用します。

## プレビュー スタイルを設定する

**パラメータ:**

- **スタイル** -- ピッカーからのプレビュー スタイル (プレビュー スタイル メニューおよびスタイル マネージャーと同じ名前)。
- **ファイル** (オプション) -- 特定のファイル。省略した場合、Marked は前面のドキュメントを使用します。ファイルがまだ開いていない場合は、Marked が最初にファイルを開きます。

スタイルを設定すると、そのテーマでプレビューが再ロードされます (プレビュー スタイル メニューからスタイルを選択するのと同じです)。

## エクスポートアクション

エクスポート アクションは同じコア オプションを共有します。

|パラメータ |説明 |
| --- | --- |
| **形式** |マークダウン、HTML、ページ分割された PDF、連続 PDF、EPUB、DOCX、ODT、RTF、RTFD、TextBundle、TextPack、または OPML。 |
| **プロフィール** (オプション) | {% prefspane Export %} から保存されたエクスポート プロファイル。 |
| **スタイル** (オプション) |エクスポート前に適用するプレビュー スタイル (完全なプレビューのリロード)。 |
| **ページ サイズ** (オプション) | `A4` や `Letter` などのページ サイズ名を印刷します。 |
| **余白** (オプション) |マージンの略記法 (以下を参照)。 |
| **フォント サイズ** (オプション) | PDF エクスポートのポイント単位の基本フォント サイズ (`14` や `12pt` など)。 |
| **出力ファイル** / **出力フォルダー** (オプション) |目的地のパス。省略した場合、Marked は正しい拡張子を使用してソース ファイルの横に書き込みます。 |

**注意事項**

- **ページ分割された PDF** は、{% appmenu File, Export As, Save PDF (Paginated) %} と同じ印刷レイアウト パイプラインを使用します。未処理の HTML プレビュー ドキュメントには使用できません。
- **HTML** エクスポートでは、生の Markdown ソース ファイルではなく、レンダリングされたプレビュー (WebView に表示されるもの) が使用されます。
- **連続 PDF** は、現在のプレビュー WebView レイアウトをキャプチャします。
- **フォント サイズ** では、{% prefspane Export %} と同じカスタム エクスポート/印刷フォント サイズ オプションが有効になります。 Fountain ドキュメントには影響しません。

### ファイルを開いてエクスポートする

Finder ワークフローに最適: Markdown ファイルを選択し、Marked で開き、ワンステップでエクスポートします。

**パラメータ:** **ファイル** (必須)、および上記のエクスポート オプション。

使用例: Finder からファイルを取得し、選択したプロファイルとスタイルで **ページ分割された PDF** をエクスポートするクイック アクション。

### ドキュメントのエクスポート

マーク付きで**すでに開かれている**ドキュメントをエクスポートします。

**パラメータ:**

- **ファイル** (オプション) -- 開いている特定のファイル。省略した場合、Marked はフロントドキュメントをエクスポートします。
- 上記と同様に、オプションと **出力ファイル** をエクスポートします。

Marked がすでに実行中で、ファイルを再度開かずに現在のプレビューをエクスポートしたい場合にこれを使用します。

### 開いているドキュメントをエクスポートする

現在マークで開いている**すべて**のプレビュー ドキュメントをエクスポートします。

**パラメータ:**

- **形式**およびエクスポート オプション (プロファイル、スタイル、ページ サイズ、余白、フォント サイズ)。
- **出力フォルダー** (オプション) -- エクスポートされたファイルのディレクトリ。省略した場合、各ファイルはソース ドキュメントの横にエクスポートされます。

マーク付きで複数の章やメモを確認した後、バッチでエクスポートする場合に便利です。

## マージンの省略表記

エクスポート アクションで **マージン** が設定されている場合は、1 ～ 4 つの測定値を含む文字列を使用します。単位: `in`、`cm`、`mm`、`pt`、または`"` (インチ)。単位のない数値はポイントとして扱われます。

|値 |意味 |
| --- | --- |
| `1in` |すべての面 |
| `1in 2in` |上下`1in`、左右`2in` |
| `1in 2in 1in` |上`1in`、左右`2in`、下`1in` |
| `1in 2in 1in 2in` |上、右、下、左 |

これは、[AppleScript](AppleScript_Support.html#with-options-properties-record) エクスポート レコードの `margins` キーと一致します。

## ワークフローの例

### Finder ファイルを PDF に変換

1. **ファイルを開いてエクスポート**
2. **ファイル** -- 共有シートまたは Finder クイック アクションから入力します。
3. **形式** -- ページ分割された PDF。
4. **スタイル** -- オプションのテーマ (Amblin など)。
5. **プロファイル** -- オプションの保存済みエクスポート プロファイル。
6. **出力ファイル** -- オプション。ソースの横に「`filename.pdf`」と書き込む場合は空のままにします。

### Marked で開いているものをエクスポートする

1. **ドキュメントのエクスポート**
2. フロント ウィンドウを使用するには、**ファイル** を空のままにしておきます。
3. **形式**とオプションのプロファイルまたはスタイルを選択します。

### 開いているドキュメントをバッチエクスポートする

1. **開いているドキュメントをエクスポート**
2. **形式** (EPUB など) を選択します。
3. 必要に応じて、すべてのエクスポートを 1 つのディレクトリに収集するように **出力フォルダー** を設定します。

### スタイルを設定してエクスポートする (2 つのステップ)

1. **プレビュー スタイルの設定** -- スタイルを選択します (オプションで特定の **ファイル**をターゲットにします)。
2. **ドキュメントのエクスポート** -- 希望の **形式**を使用した同じファイルまたはフロント ドキュメント。

**スタイル**をエクスポート アクションに直接渡すこともできます。マークするとテーマが適用され、プレビューのリロードを待ってからエクスポートされます。

## パスとサンドボックスのエクスポート

- **出力ファイル**または**出力フォルダー**を省略した場合、マークはソースドキュメントの横に書き込みます。
- エクスポート パスが **開いているドキュメントのフォルダー内**にある場合、マークすると中間フォルダーを作成できます。
- ドキュメントのフォルダー外にエクスポートするには、マークがアクセスできる書き込み可能なパス (保存されたドキュメントの場所、セキュリティ対象のブックマーク、または [開く] ダイアログで許可したフォルダー) が必要です。

同じサンドボックスルールについては、[AppleScript サポート](AppleScript_Support.html#export-paths-and-sandboxing) を参照してください。

## レガシー `convert_to` アクション

AppleScript 辞書は、プレビューを開いていない状態で Markdown テキストまたはファイルを変換するための **`convert_to`** を公開しています。上記のネイティブ ショートカット アクションが推奨されます。これらのアクションは、ドキュメントを適切に開き、プレビューのロードを待機し、ページ分割された PDF エクスポートを非同期でサポートします。

従来のコマンドの詳細については、[AppleScript サポートのショートカットと `convert_to`](AppleScript_Support.html#shortcuts-and-convert_to) を参照してください。

## トラブルシューティング: アクションがショートカットに表示されない

ショートカット インデックス **1** バンドル識別子ごとにマークされたインストール (`com.brettterpstra.marked`)。 Xcode でビルドしたアプリである必要はなく、**最も高いビルド番号** (`CFBundleVersion`) のコピーが優先されます。

ショートカット アクション ライブラリの **アプリ** に「マーク付き」が表示されない場合:

1. ディスク上のすべてのコピーをリストします。
   「」バッシュ
   mdfind 'kMDItemCFBundleIdentifier == "com.brettterpstra.marked"'
   「」
2. 古いコピー (たとえば、デスクトップ上の古い `Marked.app` またはショートカット メタデータを**含まない** ビルドからコピーされた `/Applications` ) を削除するか名前を変更します。
3. ショートカットで使用するビルドを実行します (Xcode **Run**、または DerivedData の `.app` を直接開きます)。
4. **ショートカット** アプリを終了し、再度開きます。

ショートカットを含むビルドには `Contents/Resources/Metadata.appintents` が含まれます。次のことを確認できます。

```bash
ls "/path/to/Marked.app/Contents/Resources/Metadata.appintents"
```

起動時、登録が実行されると、Marked は Console.app に `[MKShortcuts] Registering App Intents` を記録します (macOS 13 以降)。

## デバッグ

{% prefspane Advanced %} で **デバッグ モード**を有効にします。マークされたログは、Console.app および Marked のログ ビューアのプレフィックス `[AppleScript]` を含む情報レベルでのステップをエクスポートします (エクスポート パイプラインは AppleScript と共有されます)。

## エラー

アクションが失敗した場合の一般的なメッセージ:

- マークされたドキュメントは利用できません (何も開かれておらず、ファイルが指定されていません)。
- そのファイルはマーク付きで開かれていません (**ファイルを開く** または **ファイルを開いてエクスポート** を使用してください)。
- 不明なエクスポート プロファイルまたはプレビュー スタイル名。
- プレビューのロードまたはリロードを待機中にタイムアウトしました。
- エクスポート パスの権限エラー、またはページ分割された PDF の生成に失敗しました。
