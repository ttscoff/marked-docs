# <%= @title %>

Speed Read は、フォーカスされたオーバーレイに一度に 1 単語ずつ表示する RSVP スタイルの読書モードです。

## 高速読み取りの仕組み

Speed Read は、**Rapid Serial Visual Presentation** (RSVP) と呼ばれる方法を使用します。テキストの行間で目を動かすのではなく、単語が 1 つの固定位置に表示されます。これにより、読書中に通常発生する目の動き、行の変更、後戻りが軽減され、流し読みしたり、よく知っている内容を確認したり、場所を見失うことなくテキストをすばやく移動したりするのに役立ちます。

この方法は魔法ではなく、非常に高速に理解できることを保証するものではありません。読解力は、文章の複雑さ、主題への精通度、および WPM 設定に依存します。密度の高い素材や馴染みのない素材の場合は、通常、速度をできるだけ高くするよりも、適度な速度の方が効果的です。

赤い文字は単語の視覚的なアンカー ポイントを示しており、**最適認識ポイント**と呼ばれることもあります。多くの単語では、読者は最初の文字ではなく中心よりやや左に視線を向けたときに最も効率的に単語を識別します。 Speed Read は、アンカー ポイントを同じ場所に保持して強調表示することで、目に一貫したターゲットを与えます。その結果、単語間で焦点を合わせ直すことが少なくなり、テキストが進む際のリズムが安定します。

## オープニング速度読み取り

**プレビュー > 速度読み取り**、プレビュー ウィンドウの歯車メニューの **速度読み取り** 項目を使用するか、{% kbd ctrl opt S %} を押します。このメニュー項目は、Markdown プレビュー ドキュメント ウィンドウがアクティブな場合に使用できます (生の HTML プレビューおよびドキュメントが開いていない場合は無効になります)。

Speed Read が開くと、一時停止状態で開始されるため、再生が始まる前に方向を確認できます。

## オーバーレイ コントロール

オーバーレイが表示されると、次のキーが使用可能になります。

|ショートカット |機能 |
| :-- | :-- |
| {% kbd space %} |再生/一時停止 |
| {% kbd [ %} |段落の先頭にジャンプします (前の段落に戻るにはもう一度押します) |
| {% kbd ] %} |次の段落の先頭にジャンプ |
| {% kbd left %} |読書速度を下げる (WPM) |
| {% kbd right %} |読書速度を上げる (WPM) |
| {% kbd esc %} |終了速度読み取り |

括弧は段落ナビゲーション用にキャプチャされ、左/右矢印は速度変更用にキャプチャされるため、Speed Read が開いている間はこれらのキーはプレビュー ナビゲーションをトリガーしません。オーバーレイの左上隅にある円形の **X** ボタンをクリックして、オーバーレイを閉じることもできます。

`t`/`gg` (上)、`G`/`b` (下)、`,`/`.` (ヘッダー ナビゲーション) など、その他の通常のプレビュー ナビゲーション ショートカットは、Speed Read がアクティブな間も機能します。完全なリストについては、[プレビュー ナビゲーション](Keyboard_Shortcuts#preview-navigation) を参照してください。

目次は速読中にも使用できます。 {% kbd cmd t %} を押して開いて移動するか、{% kbd f %} を押してすぐにクイック検索に焦点を当てて文書ヘッダーに移動します。

## 選択範囲から開始

Speed Read を開始するときにプレビューでテキストが選択されている場合、再生では選択したテキストが使用されます。アクティブな選択がない場合、Speed Read はドキュメントのテキスト全体を使用します。

## スクロール位置と同期する

{% prefspane Preview %} で **速読をスクロール位置と同期** を有効にするか、速読オーバーレイのチェックボックスを使用して、プレビューと速読の位置を一緒に保ちます。

このオプションを有効にすると、文書の先頭ではなく、プレビューの上部に現在表示されているコンテンツから速読が開始されます。再生が進むと、表示された単語とともにプレビューがスクロールします。

Speed Read を閉じ、プレビューをスクロールし、オーバーレイを再度開くと、新しい表示位置から再生が開始されます。 Speed Read がすでに開いている後にオーバーレイ チェックボックスをオンにすると、再生は現在のスクロール位置にリセットされ、そこから続行されます。

## 記憶された速度

WPM 値は、{% kbd ← %} および {% kbd → %} で変更すると自動的に保存されます。選択した速度は、次回 Speed Read を使用するときに復元されます。
