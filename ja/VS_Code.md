# <%= @title %>

[Visual Studio Code][vscode] には、すぐに使える Marked は含まれていませんが、VS Code で書き続けながら、Marked で **ライブ Markdown プレビュー** のコミュニティ拡張機能を使用して、プレビュー、エクスポート、校正を行うことができます。

## クイックスタート

1. VS Code **Open in Marked** 拡張機能をインストールします (下記の [Open in Marked extension][ext] を参照)。
2. VS Code で Markdown ファイルを開きます。
3. ファイルを [マーク] に送信します --- 保存するとプレビューが更新されます。

## マークされた拡張機能で開く

[マークされた拡張機能で開く][ext] (Visual Studio Marketplace) は、**マークされた拡張機能で開く** アクションを追加します: エディターのタイトル ボタン、**{% kbd shift cmd m %}**、エディターとファイル エクスプローラーのコンテキスト メニュー、Marked のファイル ブラウザーのオプションの **フォルダーを開く**、ステータス バー インジケーター、およびオプションで開く前の自動保存。マークされたアプリがデフォルトの場所にない場合、設定を使用してそのアプリへのパスを設定できます。

I> この拡張機能はもともと Marked 2 用に公開されました。Marked 3 は同じスタイルのファイルと URL ハンドオフを使用するため、この統合は引き続き機能します。何か変更があった場合は、[拡張ページ][拡張] または作成者のリポジトリで更新情報を確認してください。

## 要件

マーク付きは macOS 上でのみ実行されます。 [マーク 3][マーク] と拡張機能をインストールし、必要に応じてマークされたアプリを **アプリ パス**で指定します。
